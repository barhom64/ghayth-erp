## المبدأ الحاكم

**نظام بروحين:** واجهة تشغيلية موحّدة + **المحرّكات القائمة تبقى** (روحان لنفس السجل، doc 25 §١١.٢).
كل نوع يبقى يمرّ على منفذه القائم — **لا نقل منطق ترحيل، لا هجرة، لا مساس بالدفتر.** الموحَّد هو **الواجهة فقط**.

## الدفعات (دفعة دفعة — PR + مجلس لكل دفعة)

### دفعة ١ (تشغيلية — صفر مساس بالدفتر) ← **الحالية**
**شريط أنواع موحّد (تبويبات) + توحيد القشرة** عبر الصفحات الثلاث:
- مكوّن مشترك `FinanceCreateTabs`: تبويبات (قبض/صرف · فاتورة مبيعات · فاتورة مشتريات · قيد) أعلى **كل** صفحة، يُبرز النشط، ينتقل عند النقر — فتُعرَض الصفحات كسطح تبويبي واحد.
- شريط «ابدأ من مستند» (OCR + استيراد) يُضاف لصفحتي الفاتورة (مطابقة الواقعة).
- لافتة اتجاه ثابتة على صفحتي الفاتورة (مبيعات = قبض أخضر · مشتريات = صرف كهرماني) بنفس نمط الواقعة.
- يحلّ محلّ روابط «أنواع أخرى» النصّية في صفحة الواقعة.
- **لا تغيير منافذ، لا منطق أعمال، لا هجرة.** قابل للتراجع.

### دفعة ٢ (تشغيلية) — توحيد المرفقات
توحيد مكوّن المرفقات حيث يسمح الـbackend (الواقعة + مشتريات على لوحة موحّدة؛ المبيعات تحتاج دعم backend → دفعة لاحقة بإذن).

### دفعة ٣ (تشغيلية) — توحيد توسعة السطر + المصطلحات
نمط واحد لتوسعة السطر (ربط/توزيع) عبر الصفحات + توحيد العناوين والمصطلحات.

### دفعة ٤ (معمارية — تمسّ الدفتر/الهجرة → تصعيد لإبراهيم)
الدمج الكامل في صفحة واحدة بنموذج بيانات واحد (إن قرّره إبراهيم) — هجرات + assertion على سطور القيد + اعتماد إنساني.

## الحوكمة
- ⛔ لا دفعة تمسّ الدفتر بلا اعتماد إبراهيم صريح + assertion tests.
- ✅ كل دفعة: typecheck + lint + guard + مجلس «يُعتمد» قبل «تم».
- ✅ صفر كتابة خارج مسار المالية.
