# خطة: نمذجة صفات الموظف واشتقاق قواعد الرواتب منها

**التاريخ:** 2026-07-01 · **الموجِّه:** إبراهيم («يجب أن يستوعب النظام موظف سعودي/مقيم/ذكر/أنثى… ولكلٍّ بيانات مختلفة»)
**المسار المالك:** الموارد البشرية (قائدة في الرواتب) · المالية خادمة (قيد المسيّرات فقط).

## المشكلة الجذرية (مؤكَّدة بالكود)
`employees.isNonResident` (هجرة 233) يخلط محورين مستقلّين:
- **الجنسية** (سعودي/غير سعودي) — تحكم GOSI.
- **الإقامة** (مقيم/غير مقيم) — تحكم WHT.

الرواتب (hr.ts:3374-3378) تطبّق GOSI **بنِسبة موحّدة على الجميع** بلا تفريع جنسية → غير السعودي يُخصم منه GOSI موظف (~10%) ويُحمَّل على الشركة (~12%)، والصحيح: **غير السعودي = 0% موظف + 2% شركة (أخطار مهنية فقط)**. خطأ دفتر متكرّر على كل مسيّرة.

## الصفات المطلوبة على الموظف (المحاور)
| الصفة | القيم | تقود |
|---|---|---|
| `nationality` | `saudi` / `non_saudi` (لاحقًا: `gcc`) | GOSI · السعودة |
| `residencyStatus` | `resident` / `non_resident` | WHT (يُفصل عن الجنسية) |
| `gender` | `male` / `female` | إجازة أمومة · سن تقاعد · وزن السعودة |
| (لاحقًا) `dateOfBirth`, `maritalStatus` | — | التقاعد · البدلات |

## القواعد المشتقّة
- **GOSI** على (أساسي + سكن)، سقف 45,000، نِسَب قابلة للضبط في `system_settings`:
  - سعودي: موظف **9.75%** + شركة **11.75%**.
  - غير سعودي: موظف **0%** + شركة **2%** (أخطار مهنية).
- **WHT**: على `residencyStatus = non_resident` (منطق قائم، يُعاد ربطه بالإقامة لا بالجنسية المخلوطة).
- **(لاحقًا)** الجنس: أمومة (أنثى) · سن تقاعد · وزن نطاقات السعودة.

## الهجرة المطلوبة (تحتاج إذنك — القاعدة ٢)
`ALTER TABLE employees ADD COLUMN IF NOT EXISTS`:
- `nationality varchar(20) DEFAULT 'saudi'` + CHECK(`saudi`/`non_saudi`/`gcc`)
- `gender varchar(10)` + CHECK(`male`/`female`)
- `residencyStatus varchar(20) DEFAULT 'resident'` + CHECK(`resident`/`non_resident`)
- **تعبئة تلقائية آمنة:** `nationality = CASE WHEN "isNonResident" THEN 'non_saudi' ELSE 'saudi' END`، و`residencyStatus = CASE WHEN "isNonResident" THEN 'non_resident' ELSE 'resident' END` (من الإشارة الوحيدة القائمة، ثم تصحيح يدوي لاحق). `gender` يبقى فارغًا حتى يُدخَل.
- idempotent · لا حذف · عزل مستأجر (companyId قائم على الجدول).

## الدفعات (كل دفعة: PR مستقل + assertion على سطور القيد للدفتر + بوابة المجلس)
- **دفعة ١ (عاجلة · دفتر):** الهجرة (الصفات) + GOSI يتفرّع على `nationality` في hr.ts + `postPayrollRunGL`. assertion ديناميكي يُثبت: مسيّرة فيها سعودي (9.75/11.75) وغير سعودي (0/2) → GOSI موظف=0 وشركة=2% لغير السعودي، والقيد متوازن. **قرار إبراهيم مطلوب قبل البدء (هجرة + دفتر).**
- **دفعة ٢ (واجهة):** نموذج إنشاء/تعديل الموظف يلتقط الجنسية/الإقامة/الجنس (إلزام + عربي)؛ فصل الإشارة القديمة `isNonResident` تدريجيًّا.
- **دفعة ٣+ (قواعد الجنس/الإقامة):** إجازة أمومة، سن تقاعد، وزن السعودة، إعادة ربط WHT بالإقامة.

## قرار مطلوب من إبراهيم
1. الموافقة على **الهجرة** (الأعمدة الثلاثة أعلاه + التعبئة التلقائية).
2. تأكيد **نِسَب GOSI**: سعودي 9.75/11.75 · غير سعودي 0/2. (أم نِسَب أخرى؟)
3. تأكيد نطاق الصفات (هل نبدأ بالثلاثة: جنسية/إقامة/جنس؟).

عند «نعم» أبدأ الدفعة ١ فورًا (هجرة + GOSI متفرّع + assertion + مجلس)، دون دمج تلقائي.
