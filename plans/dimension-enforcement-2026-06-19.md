# خطة تعميم التوجيه: «كل ترحيل محاسبي يحمل الأبعاد التحليلية»

- **التاريخ:** 2026-06-19
- **المدير:** الجلسة الرئيسية (Claude) — وضع تعميم التوجيهات
- **الاعتماد البشري:** «اعتمد» (إبراهيم) لبدء الدفعة.
- **التصنيف:** يمس الدفتر المحاسبي (المادة 16، 21) → كل دفعة تُحجز لاعتماد إبراهيم + assertion tests.

## 1) المواصفة

- **القاعدة:** كل سطر قيد على حساب من فئة مُبعّدة يجب أن يحمل بُعده التحليلي (مركبة/عقار/مشروع/مورد/عميل) قبل الترحيل.
- **علامة الانطباق:** كود الحساب يطابق `DIMENSION_ENFORCEMENT_RULES` (gl/ledgerTruth.ts).
- **علامة المخالفة:** الحساب مُبعّد + عمود البُعد المقابل فارغ.
- **شكل الموضع المُصحَّح:** القاعدة على `mode: "enforce"` → الترحيل يُرفض إن غاب البُعد.
- **العيوب كقيود:** الترقية إلى enforce **تُرفض القيود الإنتاجية المشروعة** إن كانت تُرحّل اليوم بلا بُعد → شرط الترقية: **تسريب = صفر** لذلك الصنف (تقرير `/reports/ledger-truth`، يحتاج DB).

## 2) المسح (قراءة فقط) — النتيجة

البنية **مكتملة ومُنفَّذة وفي main**، لا تُبنى من الصفر:

| المكوّن | الحالة |
|---|---|
| العقد النقي `assertDimensionContract` | موجود (financePostingPolicy.ts) |
| المُنسِّق `assertLedgerTruth` | موجود، مُوصَّل عند **بابَي الترحيل** (gl/posting.ts:170 · businessHelpers.ts:689) |
| الـ ratchet `DIMENSION_ENFORCEMENT_RULES` | موجود (gl/ledgerTruth.ts) — flip سطر واحد لكل صنف |
| التغطية الاختبارية | 4 ملفات (dimensionContract · ledgerTruthContract · ledgerTruthHeuristic · financialClosureRegression) |
| عدم القابلية للتجاوز | مضمونة بثلاثية الدفتر (#2660): كل INSERT للدفتر في 3 ملفات تستدعي assertLedgerTruth |

**الحالة الراهنة للإنفاذ:** `enforce` لوقود المركبة (5510) فقط؛ بقية الأصناف `warn`.

## 3) الخطة — دفعات الترقية (warn → enforce)، صنف لكل دفعة

| الدفعة | الصنف | الأكواد | بوابة الأمان قبل الدمج |
|---|---|---|---|
| **B1** | بقية المركبة | `55xx` + `5710` | تسريب مركبة = 0 (المسار الآلي fleetEngine يضع vehicleId) |
| B2 | العقار | `56xx` | تسريب عقار = 0 |
| B3 | المشروع | `5130` · `4140` | تسريب مشروع = 0 |
| B4 | المورد (عام) | `211[1-3]` | تسريب مورد = 0 (سيناريو الفاتورة مُنفَّذ أصلًا) |
| B5 | العميل | `113[1-3]` | تسريب عميل = 0 |

- كل دفعة = **flip سطر واحد** في `DIMENSION_ENFORCEMENT_RULES` + assertion tests (يرفض بلا بُعد / يقبل ببُعد) + حكم المجلس.
- **بوابة الدفتر (إبراهيم):** قبل دمج أي دفعة، تأكيد أن تقرير `/reports/ledger-truth` يُظهر **0 تسريب** لذلك الصنف على الشركات الحيّة — وإلا فالترقية ترفض قيودًا مشروعة (ارتداد stop-ship).

## 4) القرار المطلوب من إبراهيم (لكل دفعة)

نعم/لا فقط، لا قراءة كود:
1. شغّل `/reports/ledger-truth` للصنف المستهدف.
2. تسريب = 0 → «نعم، ادمج الدفعة».
3. تسريب > 0 → إمّا إصلاح مسار الإدخال أولًا، أو «افرض رغم ذلك» (قرار صريح بتحمّل رفض تلك القيود).

## 5) الدفعة الحالية: B1 (المركبة الكاملة)

- التغيير: سطر المركبة `warn` → `enforce` في `gl/ledgerTruth.ts`.
- assertion tests: 5520/5710 بلا vehicleId يُرفض؛ مع vehicleId يقبل؛ العقار/المشروع/العميل ما زالت warn.
- **محجوزة عن الدمج** حتى تأكيد إبراهيم لتسريب المركبة = 0.
