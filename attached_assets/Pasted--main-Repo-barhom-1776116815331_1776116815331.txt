## 📋 الأمر اللي تبعثه للمبرمج

انسخ هذه الرسالة كاملة:

```
السلام عليكم،

على فرع main في https://github.com/barhom64/ghayth-erp تم دفع
سلسلة commits لتوحيد النظام بدون كسر. آخر commit: f7152fb.

المطلوب منك:
────────────────────────────────────────────────────────────

1) اسحب آخر تحديث:
   git checkout main
   git pull origin main

2) ثبّت dependencies (في حال تغيّر شي):
   pnpm install

3) تحقّق أن كل شيء يمرّ:
   pnpm -r typecheck                                   # 11 workspaces
   pnpm --filter @workspace/api-server lint            # 74 ملف، 56 صلاحية
   pnpm --filter @workspace/api-server test            # 107 اختبار
   ملاحظة: الـ tests تحتاج DATABASE_URL=<قاعدة بياناتك>

4) شغّل الخادم والـ frontend:
   pnpm --filter @workspace/api-server dev
   pnpm --filter @workspace/ghayth-erp dev

5) جرّب هذه الصفحات في الـ browser وشوف الـ header الجديد
   والـ breadcrumbs الموحَّدة:
   /hr              (الموارد البشرية)
   /hr/leaves       (طلبات الإجازات)
   /crm             (إدارة علاقات العملاء)
   /fleet           (إدارة الأسطول)
   /properties      (الوحدات العقارية)
   /projects        (إدارة المشاريع)
   /legal           (الشؤون القانونية)
   /support         (الدعم الفني)
   /warehouse       (إدارة المستودعات)
   /finance/invoices (الفواتير)
   /finance/expenses (المصروفات)

6) جرّب أكشن فيه خطأ (مثلاً Cancel لإجازة بدون سبب) —
   لازم تظهر رسالة واضحة بـ field + fix بدل "حدث خطأ" عامة.

7) افتح endpoint جديد لرؤية حالة قاعدة البيانات مقابل الكود:
   GET /api/health/schema
   يرجع:
   - status: ok | degraded | critical
   - الجداول المفقودة لكل module (HR, Finance, Fleet، إلخ)
   - آخر migration طبّق

────────────────────────────────────────────────────────────

ما اللي تم فعلياً (اختصار سريع):

أ) Safety net — 5 typed error classes + PageErrorBoundary +
   /health/schema endpoint + 19 unit test جديد

ب) 6 Primitives موحَّدة جديدة:
   - components/page-shell.tsx           (PageShell + PageSection)
   - components/form-shell.tsx           (FormShell + 9 field primitives)
   - components/page-status-badge.tsx    (central STATUS_MAP)
   - components/data-table-presets.tsx   (column builders)
   - lib/api.ts                          (ApiError class + typed mutations)
   - hooks/use-lifecycle-action.ts       (shared lifecycle hook)

ج) 11 route file في backend تحوّلت من res.status(404).json() إلى
   throw new NotFoundError() — عبر 9 domains (~130 موقع)

د) 11 صفحة hub + subpage على الـ frontend الآن تستخدم
   <PageShell> + <PageStatusBadge> بدل الـ header المخصّص و
   StatusBadge القديم

كل الـ legacy القديم يشتغل كما هو — مافيه شيء معطَّل. الموحَّد
الجديد يعيش جنباً إلى جنب حتى نستبدل كل شيء في Phase 5.

مرجع الخطة الكاملة:
docs/UNIFICATION_PLAN.md
────────────────────────────────────────────────────────────

ابلِّغني إذا كل شيء يشتغل عندك أو إذا فيه صفحة معيّنة تتصرف
غلط، عشان أكمل باقي الخطة (P2 schema + P5 cleanup + P6 lint rules).

شكراً.
```

### ملاحظات مهمة للمبرمج

1. **الـ auto-sync من Replit**: إذا مفعَّل لديه، احرص أن يسحب `git pull origin main` قبل أي تعديل محلي، وإلا قد يكتب فوق التعديلات بـ snapshot أقدم (حصل معي مرتين خلال هذه الجلسة).

2. **main = فرع العمل**: لا تستعمل فروع فرعية لهذه الجلسة. كل شيء على `main` مباشرة.

3. **بعد اختبارك**: قل لي النتيجة ونكمل مع:
   - **P2**: schema consolidation
   - **P5**: حذف القديم بعد التحقق من التبنّي
   - **P6**: lint rules تمنع الرجوع للنمط القديم
