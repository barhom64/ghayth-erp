# القواعد والسياسات التشغيلية
# Rules & Policies

> **المرجع:** #1605 (تحت #1594). المبدأ الحاكم: **أي منطق تشغيلي يمرّ عبر المحركات الموجودة، لا يُبعثر داخل routes، ولا يُبنى محرك قواعد جديد.**

## المحركات الحاملة للقواعد (كلها موجودة ومُفعّلة)
| المحرك | الدور | الحالة |
| --- | --- | --- |
| `systemGovernor` | حُرّاس عامة fail-closed عند نقاط القرار | ✅ مفعّل — 6 حُرّاس |
| `workflowEngine` | الموافقات متعددة الخطوات + SLA/تصعيد | ✅ مفعّل — 9 أنواع |
| `obligationsEngine` | متتبّع مهل عام (فواتير/عقود/جلسات/وثائق) + ماسح كل ساعة | ✅ مفعّل |
| `disciplineEngine` | تحويل مخالفات HR → مواد لائحية → جزاءات (حتمي) | ✅ مفعّل |
| `autoViolationEngine` | كشف تأخير/غياب يومي (cron) → مخالفات + مذكرات | ✅ مفعّل |
| `proactiveEngine` | قواعد استباقية → مهام/تنبيهات (انتهاء عقود/تأمين/تحصيل) | ✅ مفعّل |
| `policyEngine` | فصل المهام (SoD) + الحد الأقصى للصلاحيات | ⚠️ **تدقيق فقط** — بلا إنفاذ وقت الطلب |
| `notificationEngine` | تنفيذ الإشعارات/التصعيد (قنوات متعددة + DLP) | ✅ مفعّل |

## حُرّاس systemGovernor الستة
`system_stop` (الزر الأحمر) · `company_active` · **`financial_period`** · `trial_limits` · `posting_failures_threshold` · `audit_violations`. كلها fail-closed (الخطأ يمنع العملية).

### إثبات حيّ (الحارس المالي)
بعد إغلاق الفترة المالية، أي إجراء يؤثّر على دفتر الأستاذ في تاريخها يُرفض:
```
code: SYSTEM_GUARD_BLOCK
guardName: financial_period
reason: الفترة المالية مغلقة — لا يمكن ترحيل قيود في تاريخ 2026-06-06
```
(انظر `scripts/verify-finance-posting-journey.sh`، الخطوة 6.)

## مصدر القيد المحاسبي (كل قيد له مصدر)
الترحيل يمرّ حصرًا عبر `accounting-engine` + `eventBus` (لا route خارج المالية يكتب في `journal_entries` — مؤكَّد بـ `audit:boundaries`). المصادر: `invoice` · `payroll` · `voucher` · `purchase` · `rent` · `umrah` · `manual`.

## قواعد HR (عبر المحركات لا الـroutes)
الحضور/الغياب/التأخير → `autoViolationEngine` (كشف يومي) → `disciplineEngine` (مادة + جزاء) → `notificationEngine` (إشعار/تصعيد) → أثر في الراتب عند الاعتماد. الإجازات: رصيد/تعارض/اعتماد مدير عبر `workflowEngine`.

## متابعة (موثّقة)
- **#1605 الأساسي المتبقي:** إضافة **طبقة إنفاذ `policyEngine` وقت الطلب** (حاليًا دوال تدقيق تُستدعى يدويًا من admin فقط). التعريفات جاهزة (قواعد SoD، الحد الأقصى للصلاحيات، العمليات الحسّاسة، استراتيجيات الأدوار) وتنقصها middleware ترفض الانتهاك في خط الطلب.
