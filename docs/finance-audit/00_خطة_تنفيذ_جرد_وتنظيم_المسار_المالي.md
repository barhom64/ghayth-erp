# 00 — خطة تنفيذ جرد وتنظيم المسار المالي

> المهمة الحاكمة: #2083 (FINANCE-AUDIT) · المراجع: #1715 (محرك المالية الموحد) · #1945 (تنظيف الواجهات والنموذج المتدرج) · #2066 (العقيدة الحاكمة) · #2078 (الجرد العام — هذه المهمة **مالية فقط**).
> وثيقة العقيدة تتقدم عند أي تعارض: `غيث_العقيدة_الحاكمة_والمكتبات_المشتركة.md` · المرجع التجاري: `GHAYTH_BUSINESS_AND_FINANCE_MODEL.md`.

## 1) الهدف
تحويل المسار المالي إلى محرك مؤسسي محكوم: التوجيه المحاسبي، مصدر المال، مراكز التكلفة، الحالات الثلاث، الاعتماد، القيد، التقارير، الطباعة، الأثر التشغيلي، المهام — **عبر المحرك المركزي حصرًا**، بلا تكرار ولا حقول عديمة الأثر ولا ترحيل بلا قيد متوازن.

## 2) القاعدة الذهبية (ترتيب كل شاشة مالية)
S0 السياق النشط → S1 نوع العملية → S2 المجال → S3 السيناريو → S4 الكيان → S5 البنود → S6 مصدر المال/الذمة → S7 الحساب التلقائي (**المقترح = الفعلي**) → S8 مركز التكلفة → S9 معاينة الأثر → S10 الاعتماد → S11 الدفع → S12 الترحيل → S13 الأثر التشغيلي → S14 المهام → S15 Audit/Event.
لا حقل قبل معناه (R-003)، والعرض حسب السياق لا ثابت (مبدأ §5 الحاكم من المالك).

## 3) القواعد الإلزامية الخمس (ملزمة لكل PR)
1. **الحساب المقترح = الفعلي** — أي override بصلاحية + سبب + Audit.
2. **مصدر المال لا يغيّر حساب التحميل** — يغيّر الصندوق/البنك/العهدة فقط.
3. **فصل الحالات الثلاث** documentStatus/paymentStatus/postingStatus — لا مسودة مدفوعة، لا مرحَّل بلا قيد.
4. **لا ترحيل يُدمج بلا تحقق حي** على Postgres (`pnpm db:provision-agent` ثم اختبار يؤكد DR=CR + الحسابات + الأبعاد + الفرع/الشركة + Audit).
5. **لا قيد من مسار غير مالي** — المسارات الأخرى ترسل سياقًا/مرشحًا، والمحرك يكتب القيد.

## 5) موجات التنفيذ
- **الموجة 0 (هذا الـPR)**: توثيق فقط — الملفات 00–18 + أول 10 عناصر جرد فعلية (FA-001..010 §6) + أول 10 مهام فرعية (الملف 16). لا تغيير إنتاجي.
- **الموجة 1**: جرد القوائم/الصفحات/المطبوعات (الملفات 01، 06، 13، 14).
- **الموجة 2**: جرد المحركات والترحيل (02، 04) — حصر كل مسار يكتب قيدًا، وفحص «المعاينة=القيد» على كل نقطة.
- **الموجة 3**: الصلاحيات والتقارير (07، 08، 15).
- **الموجة 4**: التكاملات وFinance Intake (09) — النقل/العقار/العمرة/المستودع/HR/Excel/API.
- **الموجة 5**: تحويل النتائج إلى Issues صغيرة (12، 16، 18).
- **الموجة 6**: تنفيذ الحرج فقط بعد اعتماد الجرد — كل إصلاح PR منفصل بتحقق حي.
