# 07 — جرد الصلاحيات المالية (الموجة 3)

## المصادر
`lib/rbacCatalog.ts` (FINANCE_ROLES, OWNER_GM_ROLES) · `lib/rbac/authorize.ts` · `audit:stop-ship` (كل كتابة لها حارس) · `contextualRbac.ts` · field policies (maskFields).

## قائمة الفحص الإلزامية (22 صلاحية من #2083)
إنشاء/تعديل/اعتماد/إلغاء/ترحيل مصروف · إنشاء سند قبض/صرف · اعتماد/ترحيل سند · إنشاء/اعتماد/ترحيل فاتورة · فتح/تعديل شجرة الحسابات · تعديل حساب مستخدم في قيد · override الحساب · override مركز التكلفة · طباعة · تصدير · مشاهدة تقارير · إلغاء/عكس قيد · تسوية بنكية · تسوية عهدة.

## منهجية لكل صلاحية
أين تُفحص (route/middleware)؟ · هل UI فقط أم خادمي؟ (**لا اعتماد على UI**) · من يملكها (أدوار)؟ · زائدة/خطرة؟ · Audit بالسبب والفرق قبل/بعد للحساس؟

## مثبت سلفًا
- override التخصيص يتطلب `finance.allocation.override` + سبب ≥10 أحرف + سجل `allocation_override_log` (enforce gate).
- approve الفاتورة يمر بـ`authorize(amount from resource)` — حد مبلغ بالسياسة.
- ترحيل التسوية البنكية: `finance.algorithms:create` + `assertFinanceRole`.
