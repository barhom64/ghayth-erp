# 21 — Financial Control Matrix / مصفوفة التحكم المالي (توثيق فقط)

> **المرحلة:** FINANCE-CORRECTION-PHASE-2 · **المسار 4 (البند 4): Financial Control Matrix** · متابعة #2118.
> **نوع المستند: توثيقي فقط — لا كود، لا API، لا سلوك، لا بيانات.** يضبط ملكية القرار والحدود بين المالية وبقية المسارات، استنادًا إلى الكود الفعلي.

## فحص عدم التكرار والحدود
1. **ما الموجود أصلًا:** جرود سابقة (#01–#11) و#19/#20 (حصر الواجهة/التقارير). لا توجد **مصفوفة تحكم** تربط لكل عملية مالية: القائد/الخادم/مالك القرار/منشئ القيد/الحدث/الأثر/التقرير.
2. **معالجة سابقة؟** لا — هذه أول مصفوفة تحكم مالي.
3. **يخص المالية؟** نعم — تضبط حدود مسار المالية مع بقية المسارات.
4. **مسار آخر يملك القرار؟** المصفوفة نفسها تُحدِّد ذلك صراحةً.
5. **endpoint/خدمة قائمة تكفي؟** نعم — توثيق فقط، لا إنشاء.
6. **ما لن يُلمس:** أي كود/API/سلوك/بيانات.
7. **لماذا ليس تكرارًا:** عدسة «ملكية القرار + الحدود» جديدة، لا تكرر الجرود الوصفية.
8. **الأثر المالي:** لا شيء (توثيق).
9. **Audit/Event/Contract؟** المصفوفة تُسجِّل وجودها لكل عملية.
10. **معيار الإنهاء:** مصفوفة كاملة + شبهات التكرار/التداخل + إغلاق المرحلة.

## المعمارية المرصودة (أساس المصفوفة)
- **منشئ القيد مركزي وموحَّد:** كل المسارات تُرحِّل أثرها المالي عبر آلية واحدة — `financialEngine.postJournalEntry(...)` (محرّكات `src/lib/engines/*`) أو `createGuardedJournalEntry(...)` (`eventListeners.ts`) — وكلاهما يصبّ في `createJournalEntry` (businessHelpers) مع حلّ الحسابات عبر `getAccountCodeFromMapping` (الخرائط المركزية). **لا منطق GL مكرَّر في أي مسار.**
- **الحدّ:** المالية **تقود** سياسة الترحيل/الحسابات/الخرائط/القيد المتوازن (DR=CR، sourceKey، الفترة)؛ كل مسار آخر **يملك حدثه التشغيلي فقط** ويستدعي الآلية المركزية. هذا يطابق عقيدة «المالية قائدة، البقية تُرسِل حدثًا».
- **القيود المباشرة في المالية:** الفاتورة (`finance-invoices.ts`)، السندات/المصروفات/القيد اليدوي/العُهد/السلف/الأرصدة الافتتاحية (`finance-journal.ts`)، الشراء/GRN (`finance-purchase.ts` + `accounting-engine.ts`)، العكس (`/journal/:id/reverse`).

## ملكية القرار — الخلاصة الحدّية
- **المالية قائدة حصريًا في:** القيد، الترحيل، الحسابات، الخرائط (`accounting_mappings`)، توازن القيد، الفترة المالية، ميزان المراجعة والتقارير المالية، سياسة الأثر المالي.
- **المالية خادمة/مستهلكة في:** العميل (CRM)، الموظف/الرواتب/الاستحقاق (HR)، المشروع (Projects بُعدًا)، الأسطول/العقار (تشغيلًا)، الوثائق (مرفقات)، العمرة (تشغيلًا).
- **لا يوجد مسار يُنشئ منطق GL خاصًّا به** — الكل يمرّ بالآلية المركزية. ✅ (لا تكرار وظيفي).

## Audit / Event / API Contract
- **API Contract:** كل ترحيل يلتزم `sourceType`+`sourceKey` (idempotency) + `companyId`/`branchId` + DR=CR + فحص الفترة (مفروض في `FinancialEngineImpl`).
- **Event:** المسارات الخادمة تطلق أحداثًا تلتقطها `eventListeners.ts` فتُرحِّل — عقد حدث واضح.
- **Audit:** عمليات المالية المباشرة تكتب `createAuditLog`/`emitEvent` (لوحظ في clients/subsidiary-provisioning/invoices…). **فجوة مرصودة (توثيق):** بعض سندات القبض/الصرف تكتب audit جزئيًا — يُراجَع كبند تحسين لاحق، لا يُعالَج هنا.

## شبهات التكرار/التداخل المرصودة (للتوثيق فقط)
1. **SALARY-ADV (مالية) مقابل JE-LOAN (HR):** متمايزان قصدًا (سُجِّل سابقًا)؛ يجب ألا يُدمجا.
2. **COGS عبر `cogsPostedAt`** مقابل ترحيل GL — إشارتان مختلفتان بالتصميم (#20).
3. **العُهد والسلف:** تقريرها بلا محدِّد ترحيل (#20) — بند تنفيذي لاحق.
4. **لا تكرار GL** بين المسارات (الآلية مركزية).
