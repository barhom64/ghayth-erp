# 13 — خطة تنظيم القوائم والصفحات المالية (بعد جرد الموجة 1)

## التجميع المستهدف (12 مجموعة)
اليومية المالية · القبض والتحصيل · الصرف والمدفوعات · الفواتير · المشتريات · العهد · البنوك والتسويات · الأصول · الضرائب · التقارير · الإعدادات · Finance Intake.

## قواعد التنظيم
- **العرض بالسياق** (مبدأ §5): البند يظهر حين يستدعيه سيناريو/حالة، لا قوائم ثابتة شاملة.
- صفحة واحدة لكل وظيفة؛ المكرر يُدمج بعقد «قائد/خادم» وأثر مكتوب.
- كل زر اعتماد/ترحيل خلفه صلاحية خادمية (لا UI-only).
- التطبيق عبر renderer واحد يقرأ scenario-model (R-003) — لا تفرّع HTML يدوي لكل صفحة.

## مخرَج هذه الخطة (يُملأ بعد 01)
جدول: الصفحة الحالية → المجموعة المستهدفة → (تبقى/تندمج مع/تؤرشف) → الأثر → معيار القبول.
