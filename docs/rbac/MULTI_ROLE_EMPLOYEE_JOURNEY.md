# رحلة الموظف متعدد الأدوار — MULTI_ROLE_EMPLOYEE_JOURNEY

> المرحلة 3 — تنفيذ **#1413 §10/§13/§15** · 2026-05-29 · فرع `claude/ghaith-foundation-audit-wdIUf`
> **الهدف:** كيف يعمل موظف واحد بصفات متعددة عبر المسارات بحساب واحد ونظام RBAC واحد.
> **يُبنى على:** `app-context.tsx` (`selectedRole`, `setSelectedRoleKey`, `userRoles[]`) + `x-selected-role` header.

---

## 1. السيناريو (#1413 §10)

أحمد لديه ثلاثة أدوار: **موظف ذاتي**، **محاسب**، **مسؤول أسطول**.

```
يدخل المالية   → يعمل بصفة "محاسب"        (ينشئ مصروفًا)
يدخل الأسطول   → يعمل بصفة "مسؤول أسطول"   (يسجّل رحلة)
يدخل بوابته    → يعمل بصفة "موظف"          (يطلب إجازته)
```

الصفة النشطة ظاهرة أعلى الشاشة، قابلة للتبديل (إن سُمح).

---

## 2. آلية اختيار الصفة (خياران مدعومان)

| الخيار | الآلية | الحالة |
|---|---|---|
| تلقائي حسب الصفحة | الصفحة تطلب feature معيّن؛ المحرك يختار الدور المانح | يُطوَّر (اشتقاق) |
| يدوي من أعلى الشاشة | `setSelectedRoleKey` → header `x-selected-role` → حصر الصلاحيات | موجود ✅ |

`is_primary` يُفعَّل كافتراض الصفة عند الدخول (RBAC-007).

---

## 3. ربط الإدخال بصاحب الدور (#1413 §13)

كل إدخال (مالية/نقل/عمرة/عقارات) ينفّذه **الموظف صاحب الدور** عبر نفس الحساب ونفس RBAC — لا "مستخدم نظام" منفصل:

| المسار | يسجّله من لديه دور | الكيانات |
|---|---|---|
| المالية | دور مالي | فواتير/قيود/مصروفات/مدفوعات |
| النقل | دور نقل | مركبات/سائقين/رحلات/وقود |
| العمرة | دور عمرة | مواسم/وكلاء/باقات/معتمرين |
| العقارات | دور عقاري | عقارات/وحدات/عقود/تحصيل |

---

## 4. مثال رحلة كاملة (متعدد الأدوار)

```
1. مدير النظام ينشئ أحمد (إنشاء سريع) + 3 أدوار + نطاق فرع مكة.
2. أحمد يدخل → الصفة الافتراضية "موظف" → يطلب إجازة (self).
3. يبدّل إلى "محاسب" → ينشئ مصروف وقود (finance, branch=مكة).
   → التدقيق يسجّل: actor=أحمد, role=محاسب, perm=finance.expenses.create, scope=مكة.
4. لا يستطيع اعتماد مصروفه (SoD) → رسالة عربية واضحة (PermissionExplainer).
5. يبدّل إلى "مسؤول أسطول" → يسجّل رحلة (fleet, branch=مكة).
6. لا يرى رواتب الآخرين، ولا يعدّل مركبة خارج صلاحيته.
```

## 5. القرارات

- **الآلية اليدوية موجودة** — يُطوَّر الاشتقاق التلقائي + تفعيل `is_primary` + إظهار الصفة بوضوح.
- **التدقيق بالصفة** شرط لإثبات الرحلة (RBAC-001).
- **اختبارات القبول:** `RBAC_MULTI_ROLE_ACCEPTANCE_TESTS` (اختبارات 1–8) و`RBAC_FRONTEND_E2E_SCENARIOS`.
