## 2. قواعد التنظيم

1. **مطابقة ثلاثية إلزامية:** كل صفحة = route في `umrahRoutes.tsx` = ظهور في الملاحة (`navigation.registry.ts` أو `umrah-tabs-nav.tsx`). صفحة بلا route أو route بلا ملاحة = تصنيف 4 (مسار ميت) أو قرار إخفاء واعٍ يُوثَّق.
2. **الـhub القائد:** `umrah-tabs-nav.tsx` هو شريط المسار — أي صفحة جديدة تُضاف له أو يُوثَّق سبب استثنائها (صفحات التفصيل مثل `pilgrim-detail` مستثناة بطبيعتها).
3. **زر = صلاحية:** كل زر إجراء على أي قائمة يلفّه `GuardedButton`/`permission-gate` بصلاحية `umrah:<action>` الدقيقة (الملف 8). م1 يجرد الأزرار التي تظهر بلا حارس.
4. **قائمة = حالة فارغة واضحة:** كل قائمة لها empty-state يوجّه المشغّل للخطوة التالية (لا جدول أبيض صامت).
5. **لا صفحة بقرارين:** الصفحة تعرض وتُحرّر؛ القرار التشغيلي (اعتماد/ترحيل) يمر بالمحرك المختص بحدث وaudit.

## 3. مخرجات م1 لهذا الملف

- جدول 44 صفاً (31 + 13) بقالب الجرد الكامل (الملف 3).
- قائمة الصفحات المرشّحة للتصنيفات 4/6/10 بأدلتها.
- قرار لكل مرشّح في الملف 5 (حذف/دمج/نقل/إبقاء بسبب).

**معيار قبول هذا المحور:** لا صفحة عمرة بحالة «غير محسومة» — كل صفحة إما مثبتة الجدوى بدليل استخدام، أو لها قرار معتمد.
