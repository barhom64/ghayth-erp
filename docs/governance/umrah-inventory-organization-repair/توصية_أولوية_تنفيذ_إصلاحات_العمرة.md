## قواعد إدارة التدفق

1. **P0 يقطع كل شيء:** أي اكتشاف جديد بتصنيف 1/11/15 أثناء أي موجة يدخل فوراً قبل استكمالها.
2. **لا موجتين على نفس الملف:** مهمتان تلمسان `umrah-entities.ts` لا تفتحان معاً (تعارضات rebase — درس مجرَّب في سلسلة #2016–#2035).
3. **كل مهمة → PR → رقم يُبلَّغ للمالك** قبل الانتقال للتالية في نفس الموجة.
4. **بوابة بين الموجات:** لا تُفتح موجة قبل إقفال P0/P1 السابقة أو تأجيلها بقرار مالك موثّق.
5. **التأجيلات المسجّلة:** U-09 مؤجَّل عمداً (السبب أعلاه)؛ U-13 معلّق على قرار المالك بقائمة المتأثرين؛ أي تأجيل آخر يُسجَّل هنا بسببه وموعد مراجعته.

## ملخص تنفيذي للمالك

الترتيب: **أقفال الحدود (U-01/U-02) ← سلامة البيانات والأحداث (U-03/U-07/U-08) ← اكتمال الإسناد والتقارير (U-05/U-06/U-04/U-10) ← مخرجات الجرد ← التنظيم البنيوي.** المنطق: نحمي الدفتر والحدود أولاً، ثم نُكمل البيانات التي تتغذى عليها التقارير، ثم نُجمّل البنية — لا العكس.
