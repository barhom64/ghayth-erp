# قالب جرد عناصر مسار العمرة

> مرتبط بـ #2080 §«شكل جرد كل عنصر». هذا القالب **إلزامي** لكل عنصر يُجرد في م1. لا يُقبل جرد ناقص الحقول.

## القالب

```markdown
### [اسم العنصر]

| الحقل | القيمة |
|-------|--------|
| **اسم العنصر** | |
| **نوع العنصر** | صفحة / قائمة / تبويب / زر / API / محرك / تقرير / قالب / صلاحية / حالة / جدول / إجراء |

| **مستخدم فعلاً؟** | نعم (الدليل) / لا (الدليل) / غير محسوم (ما ينقص للحسم) |
| **مكرر؟** | لا / نعم — مع مَن، وأيهما الأصل |

| **يحتاج دمجاً؟** | لا / نعم — مع ماذا |
```

## قواعد ملء القالب

1. **«مستخدم فعلاً؟» تحتاج دليلاً** — أحد: ظهور في `navigation.registry.ts` أو `umrah-tabs-nav.tsx`، استدعاء API من الواجهة، صفوف فعلية في الجدول على بيئة حية، أو حركة audit حديثة. «أظن أنه مستخدم» لا يُقبل.
2. **«مكرر؟» تُحسم بالوظيفة لا بالاسم** — صفحتان باسمين مختلفين تؤديان نفس الغرض = تكرار. صفحة قائمة + صفحة تفصيل = ليستا تكراراً.
3. **«يخدم محركاً مركزياً؟»** للعناصر ذات الأثر المالي/النقلي/HR: يجب ذكر اسم الدالة في المحرك (مثل `generateSalesInvoice` في `umrahInvoicingEngine.ts` أو `requestTransport` في `umrahTransportContract.ts`). إن كان العنصر يكتب مباشرة في جداول المالية/النقل = مستقل = خلل يُصنَّف 11.
4. **«أثر بقائه/تعديله»** بصيغة تشغيلية: مَن يتأثر (المشغّل/المحاسب/الوكيل) وماذا يحدث للبيانات والتقارير.
5. **الأولوية والخطورة** حسب سلّم الملف 15 (الأولوية = ترتيب التنفيذ، الخطورة = حجم الضرر عند الترك).

| الحقل | القيمة |
|-------|--------|

| **نوع العنصر** | صفحة |

| **مستخدم فعلاً؟** | نعم — مربوطة بعدّادات `unlinkedAgentCount/unlinkedGroupCount/unlinkedSubAgentCount` (migration 279) وتظهر من شاشة الاستيراد عند وجود صفوف غير مربوطة |
