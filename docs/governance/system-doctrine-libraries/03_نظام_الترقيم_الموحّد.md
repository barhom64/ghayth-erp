# ٠٣ — نظام الترقيم الموحّد

> المصدر الوحيد: `lib/numberingService.ts`. يُمنع توليد أي رقم مستند خارجه.

## ١. القاعدة
كل مستند/كيان يحمل رقمًا متسلسلًا يُصدره `issueNumber` / `generateSequentialNumber` **بعد** نجاح الحفظ، ضمن المعاملة، **لكل شركة (وفرع عند اللزوم) على حدة**. يُمنع: `MAX(id)+1`، `Date.now()`، عدّادات يدوية، أو رقم من جسم الطلب.

## ٢. البوادئ (Prefixes)
البوادئ معرّفة مركزيًّا (مثل `NUMBER_PREFIXES` في `hrEnums` لوحدة HR، ونظائرها للوحدات). أي بادئة جديدة تُضاف للمصدر المركزي، لا تُكتب نصًّا في المسار.

## ٣. الخصائص الإلزامية
1. **تسلسل لكل شركة**: لا تتداخل أرقام شركة مع أخرى (عزل مستأجرين).
2. **ذرّية**: إصدار الرقم داخل نفس معاملة الحفظ (لا فجوة سباق).
3. **ثبات**: الرقم لا يتغيّر بعد الإصدار.
4. **بادئة + سنة/فترة + تسلسل** حسب اتفاقية الوحدة، عبر المكتبة فقط.

## ٤. الاستخدام (نمط ملزم)
```ts
import { issueNumber } from "../lib/numberingService.js"; // أو generateSequentialNumber
// داخل withTransaction، بعد INSERT الأساسي:
const docNo = await issueNumber(/* النوع/البادئة، companyId من scope */);
```
- `companyId`/`branchId` من `req.scope` فقط (راجع `10_...`).
- الترقيم خدمة، لا يُنسخ منطقه في أي مسار.

## ٥. ممنوعات
- ترقيم في الواجهة الأمامية.
- بادئة مكتوبة نصًّا داخل `routes/*`.
- رقم يُقبل من المستخدم أو يُعدَّل بعد الإصدار.
