# ٠٥ — نظام الطباعة والقوالب وعرض المرفقات

> المصدر الموحّد للطباعة: أمامًا `lib/print-client.ts`. لا نوافذ طباعة يدوية ولا HTML مبعثر.

## ١. الطباعة الموحّدة
كل مستند يُطبع عبر `print-client` بقالب موحّد يحمل: ترويسة الشركة/الفرع (من السياق)، **رقم المستند** (من الترقيم الموحّد)، **تاريخ المستند وتاريخ الإنشاء بوضوح ومنفصلين**، المحتوى، حالة الاعتماد/التوقيعات، تذييل موحّد. يُمنع بناء طباعة خاصة بكل صفحة.

## ٢. القوالب
- قالب موحّد لكل عائلة مستندات (فاتورة/سند/قيد/خطاب/محضر…)، يُغذّى ببيانات الكيان — لا قالب مكرّر لكل وحدة.
- التوحيد عبر مكوّن قالب مشترك، والاختلاف في الحقول لا في البنية (راجع خطط القوالب القائمة: `*/خطة_تنظيم_القوالب*`).
- أي قالب جديد يلتزم بالترويسة/التذييل/التواريخ/الرقم الموحّدة.

## ٣. عرض المرفقات (موحّد)
- لكل كيان **منطقة مرفقات موحّدة**: رفع/عرض/تنزيل بنفس المكوّن، مع تصنيف المرفق ومالكه.
- المرفقات تُعرض ضمن صفحة الكيان (تبويب موحّد)، لا صفحة منفصلة مكرّرة (راجع `HR_UNIFIED_PAGES_ATTACHMENTS_REVIEW`).
- صلاحية رؤية/رفع المرفق تتبع صلاحية الكيان نفسه (راجع `06_...`).

## ٤. ممنوعات
- `window.print()` يدوي أو فتح HTML خام للطباعة.
- قالب مكرّر لنفس عائلة المستند.
- طباعة بلا رقم مستند موحّد أو بخلط تاريخ الإنشاء/المستند.
- صفحة مرفقات منفصلة مكرّرة لكل كيان.
