# ٠٩ — التفعيل السريع الموزّع وقوالب الوظائف

> العقيدة: لا «عهدة/مركبة إن وجدت». **قالب الوظيفة يقرّر المتطلّبات، والنظام يوزّع الإكمال على المالكين.**
> مرجع التنفيذ التفصيلي: `docs/hr/HR_QUICK_ACTIVATION_AND_COMPLETION_FLOW.md` + `docs/hr/HR_JOB_ACTIVATION_PROFILES.md` (HR-REV-3/4).

## ١. مبدأ التفعيل السريع
المُدخِل المختصّ (HR/مدير القسم) يُدخِل **الحدّ الأدنى** فقط (اسم، هوية، **فرع**، **مسمّى وظيفي/فئة**). لا تُعرض له بقية الحقول ولا خيارات الأدوار/العهدة/المركبة سطحيًّا.

## ٢. قالب الوظيفة يولّد المتطلّبات
عند اختيار المسمّى/الفئة، يقرأ النظام **قالب الوظيفة** (job activation profile) ويُولّد **قائمة متطلّبات تلقائية**:
- هل تلزم عهدة؟ أي بنود؟ (لا سؤال «افتح عهدة» — القالب يقرّر.)
- هل تلزم مركبة/رخصة؟
- أي صلاحيات/أدوار افتراضية؟
- أي وثائق (إقامة/عقد/تأمين)؟
- متطلّبات الرواتب/البنك/WPS.
المنطق من القالب، لا من خيارات يدوية في النموذج.

## ٣. توزيع الإكمال على المالكين
كل متطلّب يُسنَد لمالكه ويظهر **في مهامّه فقط**:
| المالك | يكمّل |
|-------|------|
| الموظف نفسه | بياناته الشخصية/البنكية/المرفقات الذاتية |
| مدير القسم | التبعية/المهام/التقييم المبدئي |
| الرواتب | الراتب/البدلات/WPS |
| الوثائق | الإقامة/العقد/التأمين |
| المستودع | العهدة/الأصناف |
| الأسطول | المركبة/الرخصة |
| مسؤول الصلاحيات | الأدوار/المنح |
لا أحد يرى إلا حصّته. الموظف يبقى **«قيد التفعيل»** حتى تكتمل الحصص، ولا تظهر خيارات الأدوار/المسمّيات لموظف غير مُفعَّل.

## ٤. الحالة والقبول
- آلة حالة للتفعيل: «حدّ أدنى → موزّع → مكتمل جزئيًّا → مُفعَّل».
- لا يُعتبر الموظف مُفعَّلًا حتى تُغلَق الحصص الإلزامية لقالبه.
- كل خطوة تُسجَّل (تدقيق) وتُطلق حدثًا للمالك التالي.

## ٥. ممنوعات
- نموذج تفعيل طويل يطلب كل شيء دفعةً من مُدخِل واحد.
- خيار يدوي «عهدة/مركبة إن وجدت» (القالب يقرّر).
- إظهار خيارات الأدوار/المسمّيات لموظف غير مُفعَّل أو لغير مالك الصلاحيات.
- تفعيل بلا فرع ومسمّى محدّدين.
