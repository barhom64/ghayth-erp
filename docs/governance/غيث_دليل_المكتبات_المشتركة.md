## §0 — قراءة سريعة

### المكتبة الصحيحة لكل مهمة

| تحتاج… | استخدم |
|---|---|
| كتابة قيد محاسبي | `lib/businessHelpers.createGuardedJournalEntry()` |
| فتح فاتورة مبيعات umrah | `lib/engines/financialEngine.postSalesInvoice()` *(SLICE-B قادمة)* |
| إصدار رقم تسلسلي رسمي | `lib/numberingService.issueNumber({moduleKey, entityKey})` |
| حساب الضريبة من كود | `lib/taxCodes.computeTaxFromTaxCode({code, amount})` |
| الحصول على حساب محاسبي | `lib/businessHelpers.getAccountCodeFromMapping(companyId, op, dir, fallback)` |
| التحقق من فتح فترة | `lib/businessHelpers.checkFinancialPeriodOpen(companyId, date)` |
| فتح transaction | `lib/rawdb.withTransaction(async (client) => ...)` |
| إصدار event على eventBus | `lib/businessHelpers.emitEvent({companyId, action, entity, entityId})` |
| تسجيل audit log | `lib/businessHelpers.createAuditLog({companyId, userId, action, entity})` |
| إرسال in-app notification | `lib/businessHelpers.createNotification({companyId, assignmentId, ...})` |

## §1 — Level 0: الأساس (Foundation)

### 1.1 `lib/rawdb.ts`

**المسؤولية:** كل اتصال بقاعدة البيانات. لا تستخدم `pg` مباشرة من أي ملف آخر.

**التوقيعات الرسمية:**
```ts
rawQuery<T>(sql: string, params: unknown[]): Promise<T[]>
rawExecute(sql: string, params: unknown[]): Promise<{ affectedRows: number; insertId?: number }>
withTransaction<T>(fn: (client: PoolClient) => Promise<T>): Promise<T>
```

**القواعد:**
- ✅ كل `SELECT` يلزمه `WHERE "companyId" = $N AND "deletedAt" IS NULL` على كل soft-delete table.
- ✅ كل JOIN يلزمه `companyId` + `deletedAt IS NULL` على alias الجدول المنضم.
- ✅ كل كتابة (INSERT/UPDATE/DELETE) داخل `withTransaction`، إلا لو هي تشغيلية read-cache-only.
- ❌ ممنوع template strings فيها user input مباشر — استخدم parameters.
- ❌ ممنوع `pool.query` مباشر من routes — كله عبر `rawQuery` / `rawExecute`.

**الحارس:** `check-ghost-rows.mjs` + `check-sql-ambiguity.mjs`.

## §2 — Level 1: الخدمات العرضية (Cross-cutting Services)

### 2.1 `lib/numberingService.ts` — نظام الترقيم المركزي

**المسؤولية:** إصدار كل رقم تسلسلي رسمي في النظام.

**التوقيع:**
```ts
issueNumber(params: {
  moduleKey: string;       // "umrah" | "fleet" | "hr" | "finance" | "properties" | ...
  entityKey: string;        // "sales_invoice" | "agent_invoice" | "rental_contract" | ...
  companyId: number;
  branchId?: number;
  client?: PoolClient;       // optional — لو ضمن transaction
}): Promise<IssueResult>

type IssueResult = {
  number: string;
  sequence: number;
  scheme: string;
}
```

**القواعد:**
- ✅ كل INSERT في جدول executive (clients, invoices, contracts, …) يسبقه `issueNumber`.
- ✅ `moduleKey.entityKey` يجب أن يكون موجوداً في `numbering_schemes` table (seed migration).
- ❌ ممنوع `nextval('seq_name')` مباشر — كله عبر `issueNumber`.
- ❌ ممنوع `generateTimeRef`, `generateRef`, `Math.random` كـtemp رقم.
- ❌ ممنوع `COUNT(*) + 1` أو `MAX(...) + 1` على الـnumber column.
- ❌ ممنوع writes مباشرة على `numbering_*` tables — service فقط.

**الحارس:**
- `audit-numbering-coverage.mjs` — كل route كتابة في جدول executive يجب أن يستدعي `issueNumber`.
- `audit-numbering-service-bypass.mjs` — صفر writes مباشرة على `numbering_*`.
- `audit-numbering-schemes-vs-callers.mjs` — كل `(moduleKey, entityKey)` مذكور في الكود له seed مقابل.

## §6 — قواعد إضافة مكتبة جديدة

1. **اسأل أولاً:** هل الموجود يكفي؟ لو نعم — لا تضف.
2. **PR منفصل:** مكتبة جديدة لا تنزل ضمن PR ميزة — لها PR خاص يبرّر وجودها.
3. **سطر في هذا الدليل:** أضف dictionary entry بـ:
   - المسار، المسؤولية، التوقيع الرسمي، القاعدة، الحارس.
4. **مكان صحيح في الـLevel hierarchy:** لا تضف مكتبة في Level 2 تعتمد على Level 3.
5. **حارس CI:** ضع smoke يثبت الاستدعاء الصحيح.

## §7 — أمثلة "ماذا كان وماذا أصبح"

### مثال 1 — قبل وبعد المحرك المالي

❌ **قبل** (مكرر):
```ts
// في umrahInvoicingEngine.ts
await createGuardedJournalEntry({ ... });
await rawExecute("UPDATE umrah_sales_invoices SET journalEntryId = $1 WHERE id = $2", ...);
```

✅ **بعد SLICE-B** (موحد):
```ts
const result = await financialEngine.postSalesInvoice({
  companyId, branchId, createdBy,
  moduleKey: "umrah", entityKey: "sales_invoice",
  clientId, lines, sourceRefs,
});
// result.journalEntryId يأتي رياضياً — لا UPDATE يدوي
```

### مثال 2 — قبل وبعد للترقيم

❌ **قبل** (يكسر القاعدة):
```ts
const num = `INV-${Date.now()}-${Math.random()}`;
```

✅ **بعد** (مركزي):
```ts
const { number } = await issueNumber({
  moduleKey: "umrah", entityKey: "sales_invoice",
  companyId, branchId,
});
```

### مثال 3 — قبل وبعد للحساب المحاسبي

❌ **قبل** (literal):
```ts
const arAccount = "1210";
```

✅ **بعد** (mapping):
```ts
const arAccount = await getAccountCodeFromMapping(
  scope.companyId, "umrah_ar", "debit", "1210"
);
```
