# تفعيل الذكاء الاصطناعي (LLM) — نظام غيث

> الكود **مكتمل وموصول** (نداء Anthropic فعلي في تصنيف الاتصالات وتلخيص المكالمات،
> مع تتبّع التكلفة في `ai_request_logs`). «التفعيل» = ضبط مفتاح المزوّد فقط.
> التحليلات غير الـLLM (RFM/KPI/التنبيهات) تعمل دائمًا بلا مفتاح.

## خطوات التفعيل

1. اضبط متغيّرَي البيئة على الخادم (api-server):

   ```bash
   AI_INTEGRATIONS_ANTHROPIC_API_KEY=sk-ant-...        # مفتاحك السرّي
   AI_INTEGRATIONS_ANTHROPIC_BASE_URL=https://api.anthropic.com
   ```

   يُقرآن في `artifacts/api-server/src/lib/config.ts` → `config.ai.anthropicApiKey/anthropicBaseUrl`.

2. أعد تشغيل الخادم (المتغيّرات تُقرأ عند الإقلاع).

3. **تحقّق من التفعيل** من الواجهة: «مدير النظام ← حوكمة الذكاء الاصطناعي ← نظرة عامة ←
   بطاقة *حالة تفعيل الذكاء الاصطناعي* ← زر **اختبار الاتصال***.
   - النتيجة المتوقّعة عند النجاح: **«✓ مُفعّل ويستجيب (… مللي ثانية)»**.
   - أو عبر الـAPI مباشرةً: `GET /api/admin/ai-governance/connection-test` (يتطلّب صلاحية admin)
     يرجع `{ configured, apiKeySet, baseUrlSet, model, reachable, latencyMs, error }`.

## ما الذي يتفعّل عند ضبط المفتاح؟

| الميزة | المسار | الحالة بدون مفتاح |
|---|---|---|
| تصنيف الرسائل الواردة | `communications` (receptionCategorize) | تُتخطّى بهدوء |
| تلخيص مكالمات المقسم | `admin-pbx-control` (summarizerSummarize) | تُتخطّى بهدوء |
| صياغة الردود/الترجمة/التلخيص/التوقّع | `intelligence/ai/*` + ملعب الذكاء | يرجع «[AI غير مفعل]» |

## التكلفة والمراقبة

- كل نداء يُسجَّل في `ai_request_logs` (رموز الإدخال/الإخراج، التكلفة، المدّة).
- النموذج المستعمل: مُعرّف في `aiEngine.ts` (`ANTHROPIC_MODEL`). راجِعه إن غيّرت المزوّد.
- لوحة الحوكمة (`/admin/ai-governance`) تدير المزوّدات والموجّهات (prompts) ومراجعتها.

## ملاحظات أمان

- المفتاح **لا يُخزَّن في قاعدة البيانات ولا في الكود** — يبقى في بيئة الخادم فقط.
- مدخل ذكاء «اسأل غيث» لا يولّد SQL — يطابق نيّة من قائمة بيضاء ويشغّل استعلامًا مُدقَّقًا.
- زر «اختبار الاتصال» يرسل ping ضئيلًا (4 رموز) فقط، ومحصور بصلاحية admin.
