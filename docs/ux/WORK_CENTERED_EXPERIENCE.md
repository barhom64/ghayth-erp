# التجربة المتمحورة حول العمل — WORK_CENTERED_EXPERIENCE

> المرحلة 4 — **Ghaith Operating Foundation** (#1418) · 2026-05-29 · فرع `claude/ghaith-foundation-audit-wdIUf`
> **المبدأ:** المدخل هو "ما الذي عليّ فعله؟" لا "قائمة الصفحات". مبني على مراكز المهام/القرار/الالتزامات الموحّدة.
> **يُبنى على:** `TASK_AND_DECISION_CENTER_MODEL` + `/action-center`, `/tasks`, `/obligations` الموجودة.

---

## 1. المدخل الموحّد — مركز التحكم

عند الدخول، يرى المستخدم (مفلترًا بنطاقه):

```
مركز التحكم (/action-center)
├── مهامي        (tasks المسندة لي)
├── قراراتي      (approval_actions المنتظرة اعتمادي)
├── التزاماتي    (obligations المستحقة بموعد)
└── إشعاراتي     (notifications)
```

هذه مراكز موحّدة (لا نسخة لكل مسار) — تجمّع عبر `linkedEntityType` و`entityType`.

---

## 2. من المركز إلى السياق

النقر على مهمة/قرار يفتح **سياق الكيان** (صفحة القائد) مع الخوادم داخلها:

```
مهمة "اعتماد فاتورة #1234" → صفحة الفاتورة
   ├── تفاصيل الفاتورة (القائد: Finance)
   ├── الخط الزمني / التدقيق (خادم)
   ├── التعليقات (خادم)
   ├── المرفقات (خادم)
   ├── الاعتماد (مركز القرار)
   └── زر "اعتماد" يظهر فقط لمن يملكه وفي حالة submitted
```

---

## 3. مبادئ التصميم

1. **العمل أولًا:** أول ما يراه المستخدم هو عمله المعلّق، لا شجرة قوائم.
2. **سياق واحد للكيان:** كل ما يخص الكيان (تفاصيل + خوادم) في صفحة واحدة، لا تنقّل بين أنظمة.
3. **التالي واضح:** بعد كل إجراء، يُقترح التالي (أثر + مهمة تالية إن وُجدت).
4. **لا أنظمة منفصلة:** الخادم (تعليق/مرفق/مهمة) داخل السياق لا كنظام مستقل.

---

## 4. القرارات

- **المراكز موجودة** (`/action-center`, `/tasks`, `/obligations`) — تُستخدم وتُقوّى كمدخل أساسي.
- **يُطوَّر** ربط الحدث بمهمة تلقائية (رحلة 7 الجزئية) ليصبح المركز شاملًا.
- **يُفصَّل** توزيع المراكز حسب الدور في `USER_WORK_CENTERS`.
