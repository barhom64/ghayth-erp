# مصفوفة تقييم تجربة المستخدم لكل خدمات نظام غيث

هذه الوثيقة توسع بوابة UX لتغطي كل خدمات النظام حسب فهرس `SERVICES_INDEX.md`.

## طريقة القراءة

لكل خدمة نقيس خمسة أشياء:

1. الوصول: هل يصل المستخدم للخدمة من القائمة أو البحث أو لوحة العمل؟
2. الفهم: هل العنوان، الأزرار، الحقول، الحالات، والرسائل واضحة بالعربية؟
3. الإنجاز: هل يستطيع المستخدم تنفيذ الرحلة الأساسية بدون شرح خارجي؟
4. الأثر: هل يظهر أثر الإجراء في الحالة أو التقرير أو Audit/Event؟
5. الحدود: هل بقيت الخدمة داخل اختصاصها ولم تأخذ قرار خدمة أخرى؟

## طبقات القياس

| الطبقة | ماذا تفحص؟ |
|---|---|
| Route Smoke | فتح الصفحة، عدم وجود 404، عدم وجود أخطاء Runtime |
| Arabic RTL | العربية، الاتجاه، الجداول، النماذج، رسائل الخطأ |
| Journey UX | تنفيذ الرحلة الأساسية من البداية للنهاية |
| Effect Check | أثر في الحالة أو التقرير أو Audit/Event |
| RBAC Boundary | صلاحيات الخدمة وحدود المسارات القائدة والخادمة |
| Mobile Check | قابلية الاستخدام على الجوال |
| Performance | سرعة التحميل والاستجابة |
