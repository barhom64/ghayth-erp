# نموذج تجربة غيث الكاملة — GHAITH_FULL_EXPERIENCE_MODEL

> المرحلة 4 — **Ghaith Operating Foundation** (#1418) · 2026-05-29 · فرع `claude/ghaith-foundation-audit-wdIUf`
> **المبدأ:** تجربة عربية واضحة لغير التقني، متمحورة حول **العمل** لا الصفحات. المستخدم يرى ما يخصه فقط.

---

## 1. الفلسفة

غيث لا يقدّم "صفحات" بل **مراكز عمل**: المستخدم يدخل فيرى **ما عليه فعله** (مهام/قرارات/التزامات)، ثم ينفّذ ضمن سياق رحلته، ويرى الأثر فورًا. الصفحة وسيلة لا غاية.

```
دخول → مركز التحكم (ما عليّ؟) → سياق الرحلة (المسار القائد) → إجراء → أثر مرئي
              │                         │                              │
        مفلتر بالنطاق           الخوادم داخل السياق              حالة/تقرير/سجل
```

---

## 2. ركائز التجربة

| الركيزة | الوصف | المرجع |
|---|---|---|
| العربية أولًا | كل نص للمستخدم بالعربية الواضحة؛ لا مصطلح تقني | `ARABIC_BUSINESS_TERMS` |
| متمحورة حول العمل | المدخل مهام/قرارات لا قوائم صفحات | `WORK_CENTERED_EXPERIENCE` |
| مراكز عمل حسب الدور | كل دور له مركز عمله | `USER_WORK_CENTERS` |
| الظهور المحكوم | يرى ما يخصه فقط | `VISIBILITY_ENGINE_SPEC` |
| الخادم في السياق | الخدمة المساندة داخل القائد | `SERVICE_MODE_VISIBILITY` |
| الأزرار بالصلاحية والحالة | لا زر بلا صلاحية أو خارج الحالة | `ACTION_VISIBILITY_RULES` |
| الأثر مرئي | كل إجراء يُظهر نتيجته | `IMPACT_CATALOG` |

---

## 3. لمن نصمّم؟ (المستخدم غير التقني)

- موظف عادي، محاسب، مدير فرع، مسؤول عمرة — أغلبهم غير تقنيين.
- المبدأ: لو احتاج المستخدم شرحًا تقنيًا لفهم زر، فالتصميم فشل.
- كل شاشة تجيب: أين أنا؟ ما الذي أفعله؟ ما النتيجة؟ ما التالي؟

---

## 4. القرارات

- **يُستخدم نظام المكوّنات المشتركة الحالي** (PageShell/DataTable/FormShell/StatusBadge/Timeline…) — لا بدائل.
- **التجربة تُبنى فوق محرك الظهور** (مرحلة 1) بعد سدّ فجواته (VIS-001/002).
- **يُفصَّل في:** `WORK_CENTERED_EXPERIENCE`, `USER_WORK_CENTERS`, ووثائق `frontend/*` في هذه المرحلة.
