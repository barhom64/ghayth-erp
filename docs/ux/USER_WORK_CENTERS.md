# مراكز عمل المستخدمين — USER_WORK_CENTERS

> المرحلة 4 — **Ghaith Operating Foundation** (#1418) · 2026-05-29 · فرع `claude/ghaith-foundation-audit-wdIUf`
> **المبدأ:** كل دور يدخل على **مركز عمله** الذي يعرض ما يخصه فقط. نفس المراكز الموحّدة، مفلترة بالدور والنطاق.
> **يُبنى على:** `VISIBILITY_GOVERNANCE_MATRIX` + أقسام القائمة الثمانية (`MENU_GOVERNANCE`).

---

## 1. مركز العمل لكل دور (نماذج #1413)

| الدور | مركز عمله يعرض | لا يعرض |
|---|---|---|
| موظف عادي | طلباتي، معلوماتي، مهامي الشخصية | المالية/العمرة/النقل/الإعدادات |
| محاسب (إنشاء) | فواتير/مصروفات للإنشاء، تقاريره | اعتماد، HR، إعدادات |
| مدير مالي | كل المالية + قراراته (اعتمادات) | سياسات HR |
| مدير فرع مكة | عمل فرع مكة (موظفون/طلبات/اعتمادات) | فروع أخرى |
| مسؤول العمرة | مواسم/مجموعات/معتمرون + تقارير العمرة | مالية متقدمة (إلا بدور مالي) |
| مدير HR | موظفون/إجازات/رواتب ضمن النطاق + اعتماداته | مالية متقدمة |
| مشاهد تقارير | تقارير ضمن صلاحيته (عرض فقط) | أي إجراء |

---

## 2. تركيب مركز العمل

```
مركز عمل [الدور]
├── ملخّص (KPIs تخص الدور — kpiCard)
├── مهامي/قراراتي/التزاماتي (مفلترة)
├── إجراءات سريعة (حسب الصلاحية — موجودة في sidebar quick actions)
└── الوصول للمسارات المخوّلة فقط (canAccessModule)
```

المركز ليس صفحة جديدة لكل دور — هو **نفس المكوّنات** (action-center + module dashboards) مفلترة بـ `selectedRole` + `scope`.

---

## 3. قواعد

1. **لا مركز مبني يدويًا لكل دور** — الفلترة تنتج المركز من الصلاحيات.
2. **الصفة النشطة تشكّل المركز:** أحمد كـ"محاسب" يرى مركز المالية؛ كـ"موظف" يرى بوابته (`MULTI_ROLE_EMPLOYEE_JOURNEY`).
3. **النطاق يقيّد المحتوى:** مدير فرع يرى بيانات فرعه فقط.
4. **عربية لغير التقني:** عناوين المراكز ووصفها بالعربية الواضحة.

---

## 4. القرارات

- **يُستخدم `module-dashboards` + `action-center`** كمراكز عمل مفلترة — لا صفحات جديدة لكل دور.
- **يربط** `WORK_CENTERED_EXPERIENCE` (المدخل) و`SERVICE_MODE_VISIBILITY` (الخوادم في السياق).
- **اختبار القبول:** `USER_JOURNEY_USABILITY_SCORECARD` + `NON_TECHNICAL_USER_EXPERIENCE_TESTS`.
