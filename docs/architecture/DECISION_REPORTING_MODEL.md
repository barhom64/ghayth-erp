# نموذج تقارير القرارات — DECISION_REPORTING_MODEL

> المرحلة 6 — **Ghaith Operating Foundation** (#1418) · 2026-05-29 · فرع `claude/ghaith-foundation-audit-wdIUf`
> **المبدأ:** كل قرار (اعتماد/رفض/تجاوز) قابل للتتبّع والتقرير — الشفافية أثر.
> **يُبنى على:** `approval_actions` + `audit_logs` + `REPORTING_SERVICE_CONTRACT`.

---

## 1. الفكرة

القرارات مسجَّلة في `approval_actions` (+ التدقيق بالصفة). تقارير القرارات تجيب: **من اعتمد ماذا، بأي صفة، متى، ولماذا تجاوز الحد؟**

## 2. تقارير القرار المستهدفة

| التقرير | الغرض | المصدر |
|---|---|---|
| سجل الاعتمادات | تتبّع كل قرار | `approval_actions` + الصفة |
| الاعتمادات المتجاوزة للحد | حوكمة مالية | `rbac_approval_limits` + سبب التجاوز |
| زمن الاعتماد (SLA) | كفاءة | فرق الزمن submit→approve |
| الاعتمادات الذاتية المرفوضة (SoD) | امتثال | `sodEnforcement` blocks |
| من يعتمد ماذا (per role) | شفافية الصلاحيات | `EFFECTIVE_PERMISSIONS_SPEC` |

## 3. القواعد

1. **كل قرار مُسجَّل بالصفة** (يتطلب RBAC-001).
2. **التجاوز يحتاج سببًا** (`abacConditions`/dual control) ويظهر في التقرير.
3. **عبر محرّك BI الموحّد** — لا تقرير قرارات لكل مسار.

## 4. القرار

- **يُبنى تقرير القرارات فوق `approval_actions` + التدقيق** بعد إصلاح تسجيل الصفة (RBAC-001). مسار تطور.
