# Print Platform — Architecture Lock (Phase 0)

> Status: **Locked** as of the merge of this document.  
> Scope: every PDF, Excel, printable HTML, ZATCA e-invoice, letter, contract, label or thermal receipt generated by Ghaith ERP.  
> Owner: Print Engine v2 (`artifacts/api-server/src/lib/print/*`, `routes/print.ts`).

---

## 1. The hard rule

**All document generation inside Ghaith MUST pass through the Print Platform.**

```
SPA / page / hook ──── apiFetch("/print/render", body) ────► /api/print/render
                                                                      │
                                                                      ▼
                                                              templateResolver
                                                                + dataLoader
                                                                + adapter (a4/thermal/label/excel)
                                                                + writePrintJob (audit row)
                                                                + storePrintArtifact (optional)
                                                                      │
                                                                      ▼
                                                              { jobId, base64, mime, ... }
```

**Banned in every page, component, hook, and lib file** (enforced by `scripts/src/lint-patterns.mjs` rule `direct-pdf-generation`):

- `window.print()` outside `print-button.tsx`
- `jsPDF`, `html2pdf`, `html2canvas`, `pdfMake`, `pdfmake` imports
- Manually opening a print window with `document.write(<some-locally-built-html>)`
- Calling `xlsx.write` / `ExcelJS` from the SPA to build a workbook client-side
- `window.open(blob)` with a PDF/Excel constructed in the browser

The only legitimate browser-side use of `window.print()` is inside `print-button.tsx` — it opens a popup and drops the **server-rendered** HTML payload into it for the OS print dialog. The document itself is never built in the browser.

Why this is a hard line:

| Bypassing /print/render means losing... | Concrete impact |
|---|---|
| `print_jobs` audit row | No record of who printed what, no reprint detection |
| `watermark` ("نسخة مكررة") | Reprints look like originals → fraud surface |
| RBAC `print:<entity>:create` check | Users print docs they shouldn't see |
| Branch letterhead + ZATCA fields | Tax-invalid invoices |
| `cssOverrides` injection guard | XSS via templates |
| Rate limiter (30/min) | Server DoS or storage flood |
| UTF-8 encoding (`TextDecoder("utf-8")`) | Arabic rendered as Latin-1 (#1085) |

Any code that builds a PDF or printable doc locally is treated as a **Critical Architecture Violation**.

Tables it owns (canonical):

| Table | Purpose |

| `print_jobs` | Every render's audit row (jobId UUID, copyNumber, status, approvedBy) |
| `print_reprint_requests` | Approval workflow for reprints requiring an approver |

## 3. Document lifecycle

```
   user clicks <PrintButton/>
              │
              ▼
   POST /api/print/render  ────────► returns { jobId, base64, mime, ... }
              │
              ▼
   SPA decodes base64 → UTF-8 → popup.document.write
              │
              ▼
   popup loads → auto-print script fires → OS dialog
```

For reprints requiring approval:

```
   first render attempt  ─►  PrintApprovalRequiredError (409)
              │
              ▼
   SPA POSTs /reprint-requests  ─►  pending row in print_reprint_requests
              │
              ▼
   approver opens approval queue, hits /reprint-requests/:id/approve
              │
              ▼
   approve handler internally calls renderPrint with reprintApprovedBy = scope.userId
   updates print_reprint_requests.status='approved', sets resultJobId
              │
              ▼
   SPA can re-fetch the document via /print/jobs/:id/download (if artifact retained)
```

## 4. Output formats

| Format key | Adapter | When | Mime |
|---|---|---|---|
| `a4` | `a4Adapter` | Default for any document | `text/html; charset=utf-8` (browser prints to PDF via OS dialog) |
| `thermal_80` | `thermalAdapter` | POS receipts on 80mm thermal | `text/html` (CSS @page sized for 80mm) |
| `thermal_58` | `thermalAdapter` | POS receipts on 58mm | `text/html` (58mm @page) |
| `label` | `labelAdapter` | Barcode labels | `text/html` (label-sized CSS) |
| `excel` | `excelAdapter` | Tabular exports, reports | `application/vnd.openxmlformats-officedocument.spreadsheetml.sheet` (real xlsx via ExcelJS) |

Allowed formats per entity are declared on `entityRegistry.<entity>.print.formats` and enforced at the route layer — a 403 fires if a caller asks for a format the entity doesn't expose (e.g. `pos_receipt` + `excel`).

## 5. Template hierarchy

`templateResolver.resolveTemplate()` walks four layers, returning the first hit:

```
1. Branch override     — print_template_assignments WHERE branchId=$1
2. Company default     — print_template_assignments WHERE branchId IS NULL
3. Seeded preset       — document_templates WHERE companyId IS NULL AND presetKey='classic'
4. In-memory fallback  — BESPOKE_PRESETS[entityType] || universalFallback(entityType)
```

Layer 4 — the in-memory `BESPOKE_PRESETS` — guarantees every entityType prints **something**, even on a clean install with no DB templates seeded. The `invoice` preset (full tax-invoice layout) and `official_letter`, `umrah_statement`, `umrah_runsheet` presets live here. The universal fallback covers the long tail.

Templates can be created/updated via:

- `POST /api/print/templates` (requires `templates:write`)
- `PATCH /api/print/templates/:id`
- `DELETE /api/print/templates/:id` (soft delete, `deletedAt` is set)

Soft delete is the contract — every read query already filters `WHERE "deletedAt" IS NULL`.

## 6. Variable substitution contract

Templates use `{{path.to.field}}` plus `{{#each items}}…{{/each}}` blocks. The resolver merges:

- `{{entity.X}}` — the per-entity payload from `dataLoader`
- `{{branch.X}}` — letterhead block (companyName, branchName, address, phone, taxNumber, crNumber, logoUrl, footerText, …)
- `{{client.X}}` — buyer/customer block (when the loader provides it)
- `{{vendor.X}}` — supplier block (when applicable)
- `{{this.X}}` — loop-local for items
- `{{@index}}` — 1-based row counter inside `#each`

Automatic tokens (no template work needed):

- `{{branch.letterhead}}` — full top-of-page header
- `{{branch.footer}}` — bottom-of-page footer with VAT + CR numbers
- `{{entity.itemsTable}}` — auto-built `<table>` from `data.items`

Every substitution is HTML-escaped by default (`escapeHtml` in `variableSubstitution.ts`). Skip escaping only via the explicit `{{{raw}}}` marker, which is reserved for engine-controlled blocks.

## 7. Security contract

Lockable surfaces:

| Risk | Defense |
|---|---|
| XSS via `entityId` in `<title>` | `entityId` zod regex `/^[A-Za-z0-9_\-./]+$/`, max 64 chars |
| XSS via `entityType` in filename | `entityType` zod regex `/^[a-z][a-z0-9_]*$/`, max 60 chars |
| CSS injection (`</style><script>`) via template `cssOverrides` | Strip `</style>` in every adapter before embedding |
| Spoofed `approvedBy` in audit | `reprintApprovedBy` removed from public `/render` body; only the approve endpoint sets it from `scope.userId` |
| Cross-tenant data via `loadByTable` fallback | The fallback only fires on PG SQLSTATE `42703`; every other failure returns null |
| Audit-row deletion | None — `print_jobs` rows are append-only by design |
| Storage flood / DoS | `createPerUserLimiter` 30/min on `/render`, 60/min on `/preview` |
| Template DB bloat | `htmlContent` capped at 500_000 chars, `cssOverrides` at 100_000 |

## 7a. Granular permissions (issue #1286)

The legacy print perms bundle several capabilities under one key. `print_jobs:read` for instance lets you list jobs AND prune storage AND see the archive view AND power the diagnostics page — useful for a small team but a footgun for an "audit reader" role. Issue #1286 split each capability into its own key.

| Permission | Gates | Why this verb |
|---|---|---|
| `print:create` | `POST /api/print/render` | legacy — every render |
| `print:reprint:create` | `POST /api/print/reprint-requests` | legacy — request a reprint |
| `print:reprint:approve` | `POST /api/print/reprint-requests/:id/approve` | legacy — approve a reprint |
| `print:preview:create` | `POST /api/print/preview` (ephemeral, no audit) | produces an ephemeral doc |
| `print:download` | `GET /api/print/jobs/:jobId/download` | already an allowlisted verb |
| `print:archive:delete` | `POST /api/print/jobs/prune` (destructive) | permanently removes the blob |
| `print:verify:read` | `GET /api/print/archive/:entityType/:entityId` | lookup-only |
| `print:diagnostics:read` | `GET /api/print/jobs`, `GET /api/print/jobs.csv` | lookup-only |
| `templates:read` | template list + preview | legacy |
| `templates:write` | template CRUD | legacy |
| `print_jobs:read` | full print log | legacy (still works on every endpoint above via `requireAnyPermission`) |
| `print:<entity>:create` | per-entity render gate (defined in catalogue, not yet route-enforced) | per-entity opt-out |

Every granular route uses `requireAnyPermission(<legacy>, <granular>)` so existing roles holding the legacy keys keep working without migration — owners who want narrower roles can mint them by granting only the granular keys.

Verbs follow the `rbacConsistency` allowlist (`read | create | update | write | delete | approve | self | download`); preview becomes `:create` because it produces an ephemeral document, archive prune becomes `:delete` because it permanently removes the blob.

## 7b. Audit + event entries

Every `/render` call writes exactly one append-only row to `print_jobs` via `printJobsLogger.ts`. Schema:

| Column | Meaning |
|---|---|
| `id`, `jobId` | UUID for the rendered document — what the QR points at |
| `companyId`, `branchId`, `userId` | actor context from the JWT scope |
| `entityType`, `entityId` | what was printed |
| `templateId`, `templateVersion` | which template version was applied (or `null` for in-memory presets) |
| `format`, `paperSize` | a4 / thermal_80 / thermal_58 / label / excel |
| `copyNumber`, `isReprint` | 1 for the original, 2+ for reprints |
| `watermark` | `"نسخة مكررة"` for reprints, `null` otherwise |
| `status` | `rendered` / `failed` / `delivered` |
| `pdfStorageKey` | GCS object key (`null` after retention pruning, or for legal-hold types it stays forever) |
| `verificationCode` | 8-char human-readable code (in addition to the UUID) |
| `dataSnapshot` | JSON capture of the loader output at print time — frozen, never refreshed |
| `renderedFileHash` | SHA-256 of the bytes — proves the served artifact matches the audit row |
| `createdAt` | wall-clock at render |

Mutations: `printJobs.pdfStorageKey` is cleared by the retention runner once the blob has been pruned. No other field is ever updated. The row itself is never deleted.

Legal hold (Saudi tax/commercial-record retention, issue #1286 PR 10): documents whose `entityType` is in `LEGAL_RETENTION_ENTITY_TYPES` (`invoice`, `credit_note`, `debit_note`, `pos_receipt`, `receipt_voucher`, `payment_voucher`, `journal_entry`, `delivery_note`, `purchase_order`, `goods_receipt`, `payroll`) are **never** pruned regardless of age — `prunePrintArtifacts` skips them at both the SQL filter and a per-row guard.

Reprint events: the `/reprint-requests` flow writes to `print_reprint_requests` (request) and updates `print_jobs.isReprint` + `copyNumber` (approval). The reprint approver's `userId` is stamped via the internal `reprintApprovedBy` path — clients cannot forge it through the public `/render` body.

## 7c. Stop-Ship — no parallel printing systems (issue #1286)

Two layers of mechanical enforcement guarantee the platform is the sole printing path:

1. **`scripts/src/lint-patterns.mjs`** — rule `direct-pdf-generation` rejects PR diffs containing the banned patterns from §9.
2. **`artifacts/api-server/tests/unit/printEngine.test.ts`** walks the entire `artifacts/` tree on every CI run and fails the build if:
   - any file outside the official print module calls `window.print()`
   - any file imports `jspdf` / `pdfmake` / `html2canvas` / `html2pdf` / `puppeteer` / `react-to-print`

Allowed locations are an explicit allowlist (the print SDK itself, `print-button.tsx`, the print stylesheet, docs, and tests). To add a new printable surface, wire a server-side `dataLoader` case + render through `<PrintButton entityType=... entityId=... />`. Never call the browser's native print dialog directly.

## 9. Banned patterns (lint-enforced)

`scripts/src/lint-patterns.mjs` rule `direct-pdf-generation` fails CI if any of these appear in the SPA:

```ts
// ❌  Generating PDF in the browser
import jsPDF from "jspdf";
const doc = new jsPDF();
doc.save("invoice.pdf");

// ❌  Building HTML and printing it locally
const html = `<h1>${invoice.ref}</h1>...`;
const win = window.open("");
win.document.write(html);
win.print();

// ❌  Excel built in browser
import * as XLSX from "xlsx";
const wb = XLSX.utils.book_new();
XLSX.writeFile(wb, "report.xlsx");
```

The allowed pattern:

```tsx
// ✅  Server-rendered, audited, watermarked, RBAC-checked
<PrintButton
  entityType="invoice"
  entityId={invoice.id}
  formats={["a4", "thermal_80", "excel"]}
/>
```

Or for programmatic use:

```ts
const resp = await apiFetch<RenderResponse>("/print/render", {
  method: "POST",
  body: JSON.stringify({ entityType: "invoice", entityId: id, format: "a4" }),
});
```
