# مصفوفة المسار القائد والخادم — PATH_LEADER_SERVICE_MATRIX

> المرحلة 2 — **Ghaith Operating Foundation** (#1418) · 2026-05-29 · فرع `claude/ghaith-foundation-audit-wdIUf`
> **المبدأ:** كل رحلة لها مسار **قائد** (يملك القرار والكيان الرئيسي) ومسارات **خادمة** (تقدّم خدمة دون أن تقرر بدل القائد). الخادم يظهر **داخل سياق** القائد لا كنظام مستقل.

## 1. تصنيف المسارات

| المسار | الدور | المبرّر |
|---|---|---|
| الموارد البشرية (HR) | **قائد + أصل** | الموظف هو الأصل لكل تنفيذ في كل المسارات (#1413) |
| المالية (Finance) | **قائد + خادم عام مالي** | يقود المصروف/القيد/الفاتورة؛ ويخدم كل المسارات بترحيل GL الحاجز |
| الأسطول (Fleet) | قائد | يملك المركبة/الرحلة/الصيانة |

| العقارات (Properties) | قائد | يملك العقار/الوحدة/العقد |
| القانوني (Legal) | قائد | يملك القضية/الجلسة/الحكم |
| العلاقات (CRM) | قائد | يملك العميل/الفرصة |

## 2. مصفوفة القائد × الخادم لكل رحلة

| الرحلة | المسار القائد | المسارات الخادمة | كيف يظهر الخادم |
|---|---|---|---|
| إنشاء شركة | Foundation | الترقيم، RBAC، المالية (شجرة حسابات) | تهيئة خلفية تلقائية |
| موظف + دخول | HR | RBAC، الإشعارات، الوثائق | داخل ملف الموظف |
| مصروف → اعتماد | Finance | الاعتماد، التدقيق، الإشعارات، المرفقات | أزرار/تبويبات داخل المصروف |
| رحلة نقل | Fleet | **Finance (GL)**، Warehouse (قطع صيانة)، الإشعارات | الأثر المالي يظهر كقيد مرتبط بالرحلة |
| مجموعة عمرة | Umrah | **Finance (GL)**، الأسطول (نقل)، الوثائق، ZATCA، العمولة | الفاتورة المالية + النقل داخل سياق المجموعة |
| عقد عقاري | Properties | **Finance (GL)**، الالتزامات، الترقيم | جدول الدفعات والتحصيل داخل العقد |
| قضية قانونية | Legal | الالتزامات، الإشعارات، **Finance (تسوية)**، المهام | الجلسة/الحكم داخل ملف القضية |
| رواتب | HR | **Finance (GL ذرّي)**، RBAC، التدقيق | القيد المالي مرتبط بتشغيل الرواتب |

## 3. قواعد العلاقة قائد ↔ خادم

1. **القرار للقائد:** الخادم ينفّذ خدمة (يرحّل قيدًا، يرسل إشعارًا، يخزّن مستندًا) لكنه **لا يغيّر حالة كيان القائد** إلا بطلب من القائد.
2. **GL خادم حاجز:** المالية تخدم كل المسارات بترحيل GL؛ فشل الترحيل **يوقف** الإجراء (نمط Finance-first) ويُسجَّل في `financial_posting_failures`.
3. **ظهور داخل السياق:** صفحة الخادم لا تظهر كقسم مستقل في قائمة المستخدم غير المخوّل؛ تظهر كتبويب/زر داخل صفحة القائد (يربط `SERVICE_MODE_VISIBILITY` مرحلة 4).
4. **لا ازدواج:** الخادم واحد لكل غرض؛ ممنوع أن يبني القائد نسخته الخاصة (مهام/إشعارات/مرفقات).

---

## 4. القرارات

- **التصنيف أعلاه يُعتمَد** كمرجع لحوكمة الظهور والقوائم.
- **العمرة كقائد لكنها بلا بوابة وحدة (VIS-001):** تناقض يجب إصلاحه — العمرة قائد يملك كياناته، لذا يجب أن تُحرَس بوحدة `umrah` تمامًا كبقية القادة.
- **الخوادم العامة تظهر داخل السياق** — يُفعَّل عبر `SERVICE_MODE_VISIBILITY` (مرحلة 4) وقواعد القائمة (`MENU_GOVERNANCE` قاعدة 3).
