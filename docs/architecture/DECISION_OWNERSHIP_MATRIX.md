# مصفوفة ملكية القرار — DECISION_OWNERSHIP_MATRIX

> المرحلة 2 — **Ghaith Operating Foundation** (#1418) · 2026-05-29 · فرع `claude/ghaith-foundation-audit-wdIUf`
> **المبدأ:** لكل قرار **مالك** (الدور المخوّل) و**نطاق** و**حد** و**سياسة فصل مهام**. القرارات تمرّ عبر مركز قرار موحّد (`approval_actions`)، لا اعتماد منفصل لكل مسار.
> **يُبنى على:** `RBAC_EXISTING_ASSETS_AUDIT.md` (حدود الاعتماد + SoD) + `docs/request-approval-matrix.md` (موجود).

---

## 1. عناصر القرار

كل قرار = **(من يقرر: الدور) + (على ماذا: الميزة/الكيان) + (الإجراء: approve/reject/cancel/reopen/close) + (النطاق) + (الحد المالي) + (سياسة SoD)**.

- **الحد المالي:** `rbac_approval_limits` (max_amount + requires_dual_control).
- **فصل المهام:** `rbac_sod_rules` يمنع من أنشأ السجل من اعتماده (5 قواعد مزروعة).

## 3. قواعد ملكية القرار

1. **مركز قرار واحد:** كل قرار يُسجَّل في `approval_actions` (entityType+entityId) — ممنوع اعتماد منفصل لكل مسار.
2. **النطاق يقيّد القرار:** مدير فرع مكة يعتمد طلبات فرعه فقط (نطاق branch).
3. **الحد يقيّد القرار:** الاعتماد فوق الحد يحتاج تصعيدًا/دور أعلى (يربط `APPROVAL_POLICY_EVOLUTION` مرحلة 6).
4. **فصل المهام إلزامي حيث وُجدت قاعدة:** المنشئ لا يعتمد إن منعت السياسة (`sodEnforcement`).
5. **القرار يُسجَّل بالصفة:** الدور الذي تم به القرار يُسجَّل في التدقيق (RBAC-001 — يجب إصلاحه).

---

## 4. القرارات

- **النموذج مدعوم بالكامل** عبر `rbac_approval_limits` + `rbac_sod_rules` + `approval_actions` — يُستخدم، لا يُبنى بديل.
- **محلل التعارض** (`ROLE_CONFLICT_ANALYZER` مرحلة 3) يعرض هذه القواعد لغير التقني.
- **تسجيل صفة القرار** فجوة حرجة (RBAC-001) تُعالَج في التنفيذ.
- **التصعيد متعدد المستويات** يُفصَّل في `SLA_ESCALATION_MODEL` و`APPROVAL_POLICY_EVOLUTION` (مرحلة 6).
