# حوكمة مساعد الذكاء الاصطناعي — AI_ASSISTANT_GOVERNANCE

> المرحلة 6 — **Ghaith Operating Foundation** (#1418) · 2026-05-29 · فرع `claude/ghaith-foundation-audit-wdIUf`
> **المبدأ:** خدمات الذكاء الاصطناعي **مقيّدة بالصلاحيات** — لا تكشف ولا تنفّذ ما لا يملكه المستخدم.
> **يُبنى على:** `AI_CONTEXT_SERVICE_CONTRACT` + `lib/aiGovernance.ts` + `routes/admin-ai-governance.ts`.

---

## 1. المبادئ

1. **مقيّد بالصلاحيات:** المساعد يرى/يُلخّص فقط ما يملك المستخدم صلاحية رؤيته (نفس RBAC + النطاق).
2. **لا قرار بلا اعتماد:** المساعد يقترح، لا يعتمد؛ أي فعل ذي أثر يمرّ بمركز القرار والصلاحيات.
3. **مُدقَّق:** كل استدعاء في `ai_usage_logs` (feature/tokens/cost/user/result).
4. **محكوم بالحصص:** حصة لكل مستخدم + سقف تكلفة لكل شركة + إتاحة الميزة لكل شركة.
5. **الحقول الحساسة محميّة:** لا يكشف راتب/IBAN لمن لا يملك سياسة الحقل.

## 2. حدود المساعد

| يُسمح | يُمنع |
|---|---|
| تلخيص بيانات يراها المستخدم | كشف بيانات خارج نطاقه |
| اقتراح إجراء | تنفيذ اعتماد بلا صلاحية |
| توليد مسودّة (مراسلة/تقرير) | إرسال/ترحيل دون مرور الصلاحيات |
| الإجابة ضمن النطاق | تجاوز SoD أو الحدود |

## 3. القواعد التشغيلية

- كل استدعاء عبر `aiEngine.callAI` (لا تكامل خاص بمسار — يُتحقَّق مرحلة 7).
- سياق المساعد يُبنى من استعلامات **مفلترة بنطاق المستخدم** (نفس `buildScopedWhere`).
- الإجراءات المقترحة تظهر كأزرار محكومة (`ACTION_VISIBILITY_RULES`).

## 4. القرار

- **الحوكمة موجودة** (`aiGovernance` + الحصص + التدقيق) — تُستخدم وتُقوّى بربط سياق المساعد بـ RBAC/scope صراحةً. مسار تطور مع تحقق مرحلة 7.
