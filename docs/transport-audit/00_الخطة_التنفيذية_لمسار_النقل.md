# 00 — الخطة التنفيذية لمسار النقل

> **المهمة الحاكمة:** #2079 — مسار النقل: الجرد التشغيلي الشامل وإعادة التنظيم وإعادة النمذجة.
> **المخرج النهائي ليس كودًا** — بل جرد، تحليل، تصنيف، خطة إعادة نمذجة، وقائمة مهام تنفيذية لاحقة (الملف 18).
> **قاعدة التجميد:** يُمنع أي Refactor أو حذف أو دمج قبل اكتمال ملفات هذا المجلد ومراجعة المالك.

---

## 1. النموذج الرسمي المعتمد

```text
النقل =
  أ) رحلة ركاب   (tripFamily = passenger)
  ب) نقل حمولة   (tripFamily = cargo)
```

لا يُسمح بمسار تشغيل رئيسي ثالث. كل أنواع الخدمة (`transportServiceType`:
`cargo_load / passenger_umrah / passenger_general / equipment_rental /
internal_transfer / other`) تتفرّع داخل هذين المسارين عبر `tripFamily`
(migration 295). التأجير نوع خدمة داخل السقالة، وليس مسارًا مستقلاً.

## 2. النموذج التشغيلي الكنسي (المرجع: docs/TRANSPORT_OPERATING_MODEL.md)

```text
T0 السياق النشط → T1 نوع الخدمة → T2 المصدر المربوط (لا نص حر)
→ T3 الكيان → T4 المقاطع/النمط → T5 الاعتماد (State Machine)
→ T6 التوزيع (Dispatch) → T7 تنفيذ السائق → T8 الإغلاق التشغيلي
→ T9 الأثر المالي (Accounting Candidate فقط — لا قيد من شاشة النقل)
```

والتأجير (كنوع خدمة): `R0→R10` — عميل CRM → مركبة → فترة → بسائق/بدون
→ تسعير+وديعة → تفعيل → تسليم (R7) → إرجاع (R9) → مرشّح محاسبة (R10).

## 3. حدود الخدمة (التكامل فقط — خارج النطاق كل ما عداه)

| الجهة | الدور تجاه النقل | الحدّ |
|------|------------------|------|
| العمرة | مصدر مجموعات/برامج | النقل يستقبل ولا يملك بيانات العمرة |
| CRM | مصدر العميل | `customerId` إلزامي — لا اسم نصّي حر |
| HR | مرجع الموظفين | سائق = موظف؛ سياسة السائق ملك HR |
| المالية | مستهلك الأثر | Candidate بعد الإغلاق فقط؛ الترحيل بيد المحاسب |
| الأسطول/الصيانة | مصدر المركبة وجاهزيتها | الصيانة تقود حالة المركبة |
| الخرائط | خادم | mapsService + fallback يدوي واضح |

## 5. منهجية الجرد

- **المصدر الوحيد للحقيقة:** كود main الفعلي وقت الجرد (HEAD مُثبَّت في README).
- كل عنصر يُسجَّل بالحقول القياسية (القسم 5 من #2079): الموقع بـ`ملف:سطر`،
  المحرك المرتبط، دليل الاستخدام، التكرار/التعارض، الإجراء المقترح + سببه
  + أثره + أولويته + خطورته + معيار قبوله.
- كل ملاحظة تحمل تصنيفًا من قائمة #2079 (خلل حرج / تعارض / تكرار / ميت /
  منطق خارج المحرك / إدخال يدوي غير مبرر / UX ضعيف / نموذج غير مكتمل).
- ما لم يُتحقَّق منه بفحص مباشر يُوسَم صراحةً: «غير مؤكَّد — يحتاج فحصًا حيًا».

## 6. ما يمنع الإغلاق (منقول حرفيًا من #2079)

- مراجعة رحلة الركاب كاملة، ونقل الحمولة كاملاً، وملف المركبة، وملف
  السائق، والجدولة، والخرائط، وتكامل العمرة، وتكامل المالية كأثر فقط،
  وتكامل HR كمرجع موظفين فقط.
- لا إغلاق بملاحظات بلا خطة، ولا توصية بلا سبب، ولا إصلاح بلا معيار قبول.
