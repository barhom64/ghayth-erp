## 3. المصفوفة المستهدفة (التغييرات فقط)

| الإجراء | اليوم | المستهدف |
|---|---|---|
| اعتماد/رفض حجز | bookings:update | **bookings:approve** (action جديد) — المشغّل يحرّر، المعتمِد يعتمد |
| ready_for_invoice | cargo:update | **cargo:approve** |
| عقود التأجير | vehicles:* | **fleet.rentals:*** feature مستقلة (list/view/create/update/approve) |
| إعدادات التخطيط/الخرائط | bookings:update | **fleet.settings:update** أو قصرها على admin |
| تغيير حقول ربط المصدر | update عامة | تبقى update + **حدث تدقيق مميز** (FIX-13) |

## 4. قواعد العرض
كل زر انتقال حالة خلف `permission-gate` بالـaction المستهدف؛ الحماية النهائية خادمية دائمًا؛ كل اعتماد/تجاوز (override) يسجَّل: المستخدم/الدور/الوقت/قبل/بعد/السبب — البنية قائمة (createAuditLog + جداول overrides)، المطلوب توسيع التغطية للنقاط أعلاه.

## 5. معيار قبول الخطة
- لا انتقال اعتمادي بصلاحية تحريرية.
- التأجير مفصول feature مستقلة.
- جدول feature×action×دور محدَّث في كتالوج RBAC + اختبار يطابق المسجَّل بالمستخدَم.
