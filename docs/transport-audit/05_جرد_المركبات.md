# 05 — جرد المركبات (Vehicle Master)

> هل ملف المركبة يكفي للإسناد؟ فحص الأعمدة + النموذج + بطاقة السياق + الحراس.

---

## 1. الملف الفني (migration 262 + 295) — مكتمل البنية

| المجموعة | الأعمدة | تصل للواجهة؟ |
|---|---|---|
| قرار الإسناد | **validForPassengers / validForCargo / operationalPayloadKg** (295) | ✅ نموذج الإنشاء (سويتشات + حقول) + PATCH whitelist |

## 2. الحراس المرتبطون (المحرك يملك المنطق — سليم)
- `vehicleCapacity.ts`: حارس الحمولة (NULL = تحذير ناعم لا حظر — قرار موثَّق للسجلات القديمة) + override بجدول `vehicle_capacity_overrides`.
- `assignmentSuggestionEngine.ts`: يفلتر بـvalidFor* ويزن السعة 18% والاتفاق 8%… (8 محاور).
- `getVehicleStatusImpact` (impactPreview): معاينة أثر تغيير الحالة قبل الحفظ.

## 3. بطاقة السياق `vehicle-context-card.tsx`
- تعرض: سائق معيَّن، عداد، رحلات، تزويدات + **صف «سعة المركبة وتخصصها»** (نوع/مقاعد/حمولة تشغيلية÷حد فني/رخصة مطلوبة + شارات صالحة-للركاب/للحمولات) — يظهر فقط عند توفر البيانات (السجلات القديمة لا تُشوَّه).
- تحذيرات: صيانة متأخرة، تأمين منتهٍ/قارب، حالة غير متاحة.
- **تعارض مفردات الحالة (TA-CONF-02):** البطاقة تُلوّن بحالات `active/maintenance/retired/sold` بينما enum الجدول `available/in_use/maintenance/inspection/reserved/decommissioned` — أربع حالات من ستّ تسقط على fallback الرمادي بالاسم الإنجليزي الخام. كذلك `rental-detail.tsx` يعرض `selectedVehicle.status` خامًا بالإنجليزية. يلزم قاموس حالة موحَّد (خطة الإصلاح 14 §3).

## 4. شاشات المركبة
- `/fleet` قائمة + `vehicles-create` (مكتمل بالأقسام الفنية الثلاثة بعد #2002) + `vehicle-detail` (9 تبويبات) + `vehicle-status-change` (مع معاينة أثر).
- **فجوة تحرير (TA-GAP-06):** `vehicle-detail` يحرّر 10 حقول أساسية فقط — **الحقول الفنية الـ19 غير قابلة للتحرير بعد الإنشاء** من الواجهة (PATCH الخادم يقبلها). مطلوب تبويب «الملف الفني» قابل للتحرير.

## 5. الحكم
**ملف المركبة يكفي للإسناد بنيةً وحراسًا** (TR-006/007/008 مغلقة). الفجوتان: تحرير الملف الفني بعد الإنشاء (TA-GAP-06) وتوحيد مفردات الحالة (TA-CONF-02).
