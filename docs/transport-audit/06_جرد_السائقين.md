# 06 — جرد السائقين

> السائق = موظف (HR مرجعًا) + منفّذ محكوم بحراس + تطبيق ذاتي محظور ماليًا.

---

## 1. الهوية والجسر مع HR
- `fleet_drivers.employeeId` → جسر الموظف؛ `resolveDriverFromScope` يحلّ السائق من هوية الجلسة (لا من body) — السائق لا ينتحل سائقًا آخر.
- هوية سعودية (280): nationalId, iqamaNumber, licenseIssuingAuthority + licenseClass/Expiry.
- صفحات: drivers list (+حساب بوابة) / drivers-create / driver-detail (+تبويب «ذكاء السائق»).

## 2. الحراس (المحرك يملك السياسة)
| الحارس | الملف | المنطق |
|---|---|---|
| الأهلية | `driverEligibility.ts` | تسلسل فئات الرخصة ضد requiredLicenseClass للمركبة + override موثَّق (`driver_eligibility_overrides`) |
| الراحة | `driverRest.ts` | restHoursRequired منذ lastDutyEndedAt (تُختم من إنهاء الملاحة) |
| التخصص | `driverServiceProfile` | cargo/umrah/passenger/rental/mixed — يغذي الترتيب |
| الذكاء | `driverIntelligence.ts` | نافذة 90 يومًا: dispatchCount, start/completion/onTimeRate, avgLate, serviceMix, **reputationScore** (0.4·التزام+0.4·إنجاز+0.2·انطلاق) + specialty تلقائي ≥60% |
| في محرك الإسناد | `driverReputation` محور بوزن 10% + مضاعف تخصص ×1.2/÷0.7؛ الجديد يحصل 50 محايدًا |

## 3. تطبيق السائق (T7)
| القدرة | الموضع | الحالة |
|---|---|---|
| توافر (متاح/خارج الدوام) | `PATCH /fleet/me/availability` | ✅ |
| رحلاتي + بدء/إنهاء | `/fleet/me/trips(/:id/start|complete)` | ✅ (طبقة fleet_trips legacy) |
| بضائعي + 7 خطوات | `/fleet/me/cargo(/:id/advance)` | ✅ forward-only |
| نقاط التشغيل | `/fleet/me/cargo/:id/checkpoint(s)` | ✅ backend — **حوار SPA مركون** (TA-GAP-02) |
| ملاحة | navigation start/ping/event/complete + صفحة `/me/driver/navigation` | ✅ Phase 1 (لا turn-by-turn) |
| **حظر المال** | types تُسقط cost عمدًا (me-driver.tsx:23) + grep سالب في الاختبارات | ✅ مثبَّت |

## 4. لوحة القيادة الذكية
- `/fleet/transport/driver-intelligence`: leaderboard ببطاقات «الأفضل للعمرة/للحمولات/الأكثر إنجازًا/يتأخر كثيرًا» + جدول السمعة. (وتبويب داخل driver-detail.)
- telematics scorecard منفصل (سلوك قيادة من CMSV6).
