# 03 — جرد رحلات الركاب (tripFamily = passenger)

> تتبُّع الترتيب القاطع T0→T9 خطوةً خطوة على main الفعلي، مع موضع كل خطوة (ملف:سطح) ودليلها.

---

## T0 — السياق النشط للمدخِل
- المصدر: `app-context` + `auth` على مستوى التطبيق؛ كل راوتر نقل خلف `authMiddleware` و`scope.companyId` خادمي المصدر (لا من المستخدم).
- الحالة: ✅ منفَّذ نظاميًا.

## T1 — نوع الخدمة
- `transport-booking-create.tsx` — `SERVICE_TYPES`: cargo_load / passenger_umrah / passenger_general / equipment_rental / internal_transfer / other.
- الخادم يشتق `tripFamily` تلقائيًا (`deriveTripFamily` في transport-bookings.ts): passenger_* → passenger؛ والملتبس يُرجَّح بالبيانات (passengerCount/cargoWeight).
- الحالة: ✅. **ملاحظة UX:** الجرد القاطع يطلب «خيارين في البداية (ركاب/حمولة) ثم التفرّع» — الشاشة الحالية تعرض الأنواع الستة مسطّحة دون تفرّع هرمي (TA-UX-01).

## T2 — المصدر المربوط (لا اسم نصّي)
- `BookingSourceSelector` أعلى النموذج: عمرة (UmrahGroupPicker يملأ المجموعة/العدد) / عميل CRM / عقد / مشروع.
- إجبار خادمي: `createBookingSchema.refine` يرفض حجزًا بلا أحد (customerId / umrahGroupId / contractId / projectId / waqfId / beneficiary pair) — حقول الاسم/الجوال read-only في الواجهة + زر الحفظ معطَّل بلا مصدر.
- الحالة: ✅ (أُغلق TR-003). **بقيّة:** `updateBookingSchema.partial()` يسمح بمسح الربط على PATCH دون إعادة تأكيد — مقبول إداريًا لكنه يستحق سجل تدقيق إلزاميًا (TA-SEC-02).

## T3 — كيان الركاب
- passengerCount + umrahGroupId + flightNumber + supervisor + hotel + routeType (محددات مكة/المدينة/زيارة).
- `UmrahContextQuestionnaire` للحقول السياقية. الحالة: ✅.

## T4 — المقاطع + النوافذ الزمنية + الاتفاق
- `MultiLegBookingEditor` → `lines[]` (transport_booking_lines) مع متطلبات لكل مقطع (نوع مركبة/سعة/رخصة).
- نوافذ: pickupWindowStart/End, dropoffWindow*, fixedAppointmentTime, isFlexibleTime, priority (271).
- اتفاق العميل: requestedVehicleClass + vehicleSubstitutionPolicy (6 سياسات) + allowUpgrade + requiredExactVehicle/Driver.
- `LocationKindPicker` (مطار/بوابة/فندق/مزار/…) + geo header (279) + geo lines (292).
- الحالة: ✅ كاملة.

## T5 — الاعتماد (State Machine)
- 10 حالات بجدول انتقالات صارم في الخادم (`BOOKING_TRANSITIONS`)؛ الواجهة (booking-detail) تعرض أزرار الانتقال المسموح فقط، بعربية كاملة.
- الحالة: ✅. **فجوة سياساتية:** الانتقال approved لا يتطلب صلاحية أعلى من `fleet.bookings:update` — لا فصل بين «من يُنشئ» و«من يعتمد» (TA-PERM-01، خطة الصلاحيات 15 §3).

## T6 — التوزيع
- `POST /transport/dispatch-orders`: فحوصات صلبة — تعارض نافذة زمنية (tstzrange) للسائق والمركبة + أهلية الرخصة (`assertDriverEligibility`) + راحة السائق (`assertDriverRest`) + سعة (`assertVehicleCapacity`)؛ تجاوز موثَّق بـ`overrideReason` يُسجَّل في جداول overrides.
- اقتراح الإسناد: `AssignmentSuggestDialog` → `suggest-assignment` → محرك بأوزان 8 محاور (سعة/توفر/تعارض/راحة/رخصة/مسافة/اتفاق/سمعة السائق) **بأسباب عربية** + `suggestDiagnostics` يشرح لماذا 0 مرشّح.
- إعادة جدولة drag-and-drop: `/dispatch-orders/:id/reschedule` يعيد الفحوص.
- الحالة: ✅ قوية. (A-02 — رفض 45 راكبًا لباص 40 بشرح: الحقول والمحرك موجودان؛ **الإثبات الحيّ منوط بـE2E** — الملف 17.)

## T7 — تنفيذ السائق
- قبول/رفض/بدء/إنهاء عبر `PATCH /dispatch-orders/:id` + جلسة ملاحة (`navigation/start|ping|event|complete`) تَختم `lastDutyEndedAt`.
- عرض السائق `/me/driver`: **حظر مالي مؤكَّد** (types تُسقط cost عمدًا — السطر 23). الحالة: ✅.

## T8 — الإغلاق التشغيلي
- dispatch: completed→closed (المشغّل). booking: in_progress→completed. الحالة: ✅.

## T9 — الأثر المالي (Candidate فقط)
- **للركاب تحديدًا: لا يوجد مولِّد مرشّح محاسبة** — `createCargoBillingCandidate` (حمولة) و`createRentalBillingCandidate` (تأجير) موجودان، بينما رحلة الركاب المغلقة لا تنتج مرشّحًا آليًا؛ تمرّ عبر `transport_service_lines` يدويًا أو لا تمرّ.
- **فجوة جوهرية:** TA-GAP-01 (سجل المشاكل، أولوية عالية) — مطلوب `createPassengerBillingCandidate` على إغلاق الحجز (quantity = passengerCount, unit='pax').

## تكامل العمرة (A-01)
- `POST /transport/integration/from-umrah-group/:groupId`: يولّد 3 مقاطع قياسية (مطار→مكة، مكة→المدينة، المدينة→مطار) بعدد المعتمرين، idempotent على (group, routeType). الفندق/العدد يُسحبان بلا إدخال يدوي. ✅ — الإثبات الحيّ في E2E.
