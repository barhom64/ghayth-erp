## 1. خدمة الخرائط `lib/fleet/mapsService.ts`

| الجانب | الواقع |
|---|---|
| المزوّدون | `manual_only` (افتراضي — haversine ×1.3 معامل التفاف) · `google_maps` (حيّ عبر mapsGoogleProvider) · `mapbox` و`here_maps` **stubs غير فعّالة** |
| مصدر المفتاح | `transport_planning_settings.mapProviderApiKey` (لكل شركة) ← fallback `config.googleMapsApiKey` — **لا process.env مباشر** (التزام FND-003) ✅ |
| التدهور الرشيق | مزوّد بلا مفتاح → سقوط شفاف إلى manual_only + علم `isApproximate` («تقدير تقريبي» في الواجهة) ✅ |
| فحص الصحة | `POST /transport/planning-settings/health-check` يتحقق من المفتاح ويُرجع quota_exceeded على 403 |
| **مراقبة الحصة** | ❌ لا تتبّع استباقي ولا قاطع دارة للخرائط (TR-029) — TA-GAP-09 |
| الكاش | `transport_route_estimates`: TTL يوم، مطابقة ±0.0001 — يخفض الاستهلاك ✅ |
| المستهلكون | محرك الإسناد (محور المسافة) · estimate-route · جلسات الملاحة (ETA) |

## 2. المواقع
- `transport_locations`: code+name+إحداثيات+locationType (10 أنواع: airport/gate/hotel/mazar/warehouse/project/customer_site/depot/mosque/other) — CRUD ضمن bookings router.
- Geo مزدوج المستوى: رأس الحجز (279) + المقاطع (292) — نص + kind + lat/lng + placeId.
- `LocationKindPicker` يفرض النوع في الواجهة. **بقيّة TR-028:** حقلا from/toLocationText يقبلان نصًا حرًا إلى جانب المعرّفات — مقبول كوصف مكمل، ويُحسم في خطة 14 §5 (نص = label فقط حين يوجد locationId).
- لقطات المواقع: `vehicle_location_snapshots` (pings الملاحة) — وقود التتبع الحي.

## 3. telematics (تتبع مباشر منفصل عن الملاحة)
CMSV6: مواقع/أحداث/حساسات مستمرة + فيديو عند الطلب + قاطع دارة (`telematicsReliability`) + idempotency على externalEventId + retention 03:00. خرائط حية في `/fleet/telematics/live-map`. — ناضج، خارج نطاق إعادة النمذجة.
