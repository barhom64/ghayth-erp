# سجل قرار — سقف الوقت الإضافي الشهري القابل للتحكم

- **المعرّف:** HR-OVERTIME-MONTHLY-CAP
- **التاريخ:** 2026-06-20
- **النوع:** سياسة HR قابلة للضبط (إعداد) — لا مساس بالدفتر
- **القابلية للتراجع:** كاملة (الافتراضي = بلا حدّ → صفر تغيير للسلوك القائم)

---

## الملخص

كان للوقت الإضافي **سقف يومي** (12 ساعة) فقط، **بلا سقف شهري** — فيمكن تسجيل
ساعات إضافية شهرية غير محدودة لموظف واحد. بدل فرض رقم سياسة (يخصّ المالك)، أُضيف
**سقف شهري قابل للتحكم** عبر إعداد:

- **المفتاح:** `system_settings.overtime_monthly_cap_hours`
- **الافتراضي:** غائب / 0 = **بلا حدّ** → صفر تغيير للسلوك القائم.
- **عند الضبط > 0:** يرفض إنشاء طلب وقت إضافي إذا تجاوز مجموع ساعات الشهر
  (pending + approved، يستثني المرفوض) لنفس الموظف السقفَ — برسالة تبيّن المتبقي.

## كيف يُضبط (controllable)

عبر آلية الإعدادات العامة القائمة (صفر تكرار، بلا UI جديد للجوهر):

```
PUT /settings/   { scopeOverride: "company", key: "overtime_monthly_cap_hours", value: "60" }
```

(تُقرأ بنمط `getCompanyVatRate`: قيمة الشركة، أو القيمة العامة، أو الافتراض «بلا حدّ».)

  عند التجاوز برسالة المتبقي، بقاء السقف اليومي (12).
- `typecheck` (api-server) نظيف.

## المخاطر المتبقية

- لا مساس بالدفتر؛ الافتراضي يحفظ السلوك القائم.
- **تحسين واجهة اختياري:** حقل مُعنوَن في تبويب الضوابط التشغيلية
  (`system-controls-tab`) بدل ضبط المفتاح عبر الإعداد العام — لراحة المشغّل
  (لا يلزمه معرفة اسم المفتاح). الجوهر (التحكم + التطبيق) مكتمل.

## قرار مطلوب من إبراهيم

اعتماد السقف الشهري القابل للضبط (افتراضه «بلا حدّ»)، وإقرار تأشير حقل الواجهة
المُعنوَن كتحسين اختياري.
