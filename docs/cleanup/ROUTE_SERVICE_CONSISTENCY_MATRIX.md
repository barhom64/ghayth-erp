# Route ↔ Sidebar ↔ TabsNav ↔ Backend ↔ Owner Consistency Matrix

> **Generated by Agent 5 (Route/Sidebar/Service Consistency).** Cross-references every frontend route with its sidebar entry, tabs-nav entry, backend mount module key, service group, and owner. Companion to `docs/audit/PAGE_VISIBILITY_AND_SERVICE_EXPOSURE.md` and `docs/audit/PAGE_SERVICE_CLASSIFICATION.md`.

## Methodology

- **Route**: extracted from `artifacts/ghayth-erp/src/routes/*.tsx` (15 files).
- **Page file**: the routes file declaring the route (proxy for the page module).
- **Sidebar entry?**: at least one entry in `allNavSections` of `sidebar-layout.tsx` with this `path` (or with a query string prefix matching it).
- **TabsNav entry?**: appears as `href` or `match` in any `*-tabs-nav.tsx` under `src/components/shared/`.
- **Backend mount module**: the longest `router.use("<prefix>", requireModule(...), ...)` prefix in `artifacts/api-server/src/routes/index.ts` that matches the frontend route.
- **Service group**: the routes file owner (Finance, HR, Fleet, …). `Misc/Cross-cutting` for `miscRoutes.tsx`.
- **Owner**: derived from module → team lead mapping (Finance Lead, HR Lead, Platform Admin, …). Unassigned when no module bound.
