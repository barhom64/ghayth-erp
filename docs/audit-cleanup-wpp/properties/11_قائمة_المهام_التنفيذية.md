## T-2X — b-parts (بعد الدمج)

### T-21 b-part #2039 — تجاري VAT (راوتر + رحلة)
- **العمل:**
  1. تأكيد `POST /payments/:id/pay` يميّز commercial_rent ويستدعي split.
  2. رحلة E2E تجاري في `verify-property-rent-journey.sh` (عقد تجاري → دفع → تحقّق 3 أسطر).
- **DoD:** رحلة E2E تجاري 100% خضراء، 3 أسطر مع 2131 VAT.

### T-22 b-part #2042 — البيع (جدول + راوتر + واجهة + رحلة)
- **العمل:**
  1. ترحيل `property_sales` table.
  2. `POST /properties/sales` + Zod schema.
  3. واجهة `sale-create.tsx` ضمن `/properties/create`.
  4. رحلة E2E بيع تؤكّد DR ذمم + CR أصل + CR/DR مكسب/خسارة.
- **DoD:** بيع كامل من UI إلى GL + رحلة E2E خضراء.

### T-23 b-part #2043 — الإدارة (commissionPct + راوتر + كشف مالك)
- **العمل:**
  1. ترحيل عمود `rental_contracts.commissionPct` (default = system setting).
  2. `POST /properties/management-collection` (إن لزم — وإلا في `/payments/:id/pay`).
  3. كشف المالك يقرأ من `journal_lines`.
  4. رحلة E2E إدارة (تحصيل + كشف).
- **DoD:** كشف مالك يطابق دفتر الأستاذ + رحلة E2E خضراء.

### T-24 b-part #2044 — قرار الصيانة (ownerType)
- **العمل:**
  1. قراءة `property_buildings.ownerType` في `/maintenance-requests/:id/complete`.
  2. توجيه إلى `postMaintenanceExpenseGL` أو `postMaintenanceOwnerBillingGL`.
  3. عند billing — استدعاء `requestInvoiceCreation` لفاتورة ضريبية للمالك.
  4. رحلة E2E لكلتا الحالتين.
- **DoD:** كلا المسارين خاضع للقيد الصحيح + فاتورة للمالك في حال billing.
