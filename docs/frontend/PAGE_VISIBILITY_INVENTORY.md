## 0. الغرض من هذا الملف

غيث ليس مجموعة صفحات. هذا الملف يجرد **كيف تَظهر الصفحة وكيف تختفي** اليوم فعليًا في الكود، ويصدر قرارًا لكل آلية ظهور: تُستخدم / تُطوَّر / تُدمَج / تُخفى / تُحذف. الهدف أن يصبح الظهور **محكومًا مركزيًا** ومرتبطًا بـ (الدور + النطاق + التفعيل + حالة السجل)، لا متناثرًا.

لكل بند يجيب الجرد على: ما الموجود؟ أين؟ هل يُستخدم؟ هل صالح؟ هل مكرر؟ هل يحتاج تطوير؟ هل يجب إخفاؤه؟ هل يجب حذفه؟ القرار المقترح.

## 2. محرك الظهور الحالي — كيف تَظهر الصفحة؟

الظهور يُحسم على **طبقتين** (التوجيه + القائمة)، وكلاهما يستهلك نفس دوال `AppContext`.

### 2.1 طبقة التوجيه (Route gate) — `src/App.tsx`

كل إدخال مسار شكله: `{ path, component, subKey?, minRoleLevel? }` (`App.tsx:41-45`). ثم يُوسَم بوحدة عبر `tagRoutes` (`App.tsx:44`) ويُلَفّ بـ `ModuleRoute` (`App.tsx:104-110`) الذي يمنع العرض إذا تحقق أيٌّ من:

```
(module && !canAccessModule(module))
|| (subKey && module && !canAccessSubPage(module, subKey))
|| (minRoleLevel !== undefined && roleLevel < minRoleLevel)
```

**ثلاث بوابات متراكبة:**
1. **بوابة الوحدة** `canAccessModule(module)` — الوحدات المسندة للمستخدم.
2. **بوابة الميزة الفرعية** `canAccessSubPage(module, subKey)` — تحكم دقيق داخل الوحدة.
3. **بوابة المستوى** `roleLevel >= minRoleLevel` — مستويات 10→100.

تعيين الوسوم (`App.tsx:47-62`):

| الوحدة | `tagRoutes` | `minRoleLevel` افتراضي |
|---|---|---|
| hr, finance, fleet, property, store, documents, requests, comms | نعم | — (بوابة وحدة فقط) |
| governance | نعم | 60 |
| bi, legal | نعم | 40 |
| admin | نعم | **90** |
| settings | نعم | **70** |
| **umrah** | **لا — يُدمج خامًا** | **لا شيء** ⚠️ |
| **misc** | **لا — يُدمج خامًا** | per-entry فقط |

### 2.2 طبقة القائمة (Sidebar gate)

انظر `MENU_GOVERNANCE.md` — القائمة تُولَّد من مصفوفة `allNavSections` المعرّفة يدويًا في `sidebar-layout.tsx`، وتُفلتر بنفس دوال `canAccessModule` / `canAccessSubPage` / `roleLevel` + حقل `perm` (غير مفعّل عمليًا).

### 2.3 لا توجد بوابة "تفعيل/اشتراك" (Feature Flag) على مستوى التوجيه

بحث في كل ملفات المسارات عن `featureFlag` / `FF_` / `flagged` — **لا وجود**. أي **التفعيل الجزئي الذكي** (مبدأ غيث الأساسي: المسار القائد/الخادم، تفعيل جزئي) **غير ممثَّل في طبقة الظهور**. هذه أهم فجوة في الجرد.

## 5. خريطة الظهور المطلوبة (الهدف — لا تُنفَّذ الآن)

```
صفحة تَظهر للمستخدم  ⇔  (المسار مسجَّل في registry)
                      ∧ (الميزة مُفعَّلة للمنشأة — Feature/Subscription)   ← VIS-002 (مفقود)
                      ∧ (المستخدم يملك صلاحية feature.view ضمن النطاق)     ← يُشتق من featureCatalog
                      ∧ (الصفحة ناضجة أو المستخدم مخوّل لرؤية التجريبي)     ← VIS-004 (مفقود)
زر يَظهر داخل الصفحة  ⇔  (صلاحية feature.action ضمن النطاق)
                      ∧ (حالة السجل تسمح بالإجراء)                          ← VIS-005 (مفقود)
```

يُفصَّل هذا في المرحلة 4: `VISIBILITY_ENGINE_SPEC.md` و`ACTION_VISIBILITY_RULES.md` و`SERVICE_MODE_VISIBILITY.md`.
