## 2. منطق فلترة الظهور (`filterItems` — `sidebar-layout.tsx:733-758`)

```
لكل عنصر:
  if (item.module && !canAccessModule(item.module)) → إخفاء          // بوابة الوحدة
  if (item.minRoleLevel && effectiveRoleLevel < item.minRoleLevel) → إخفاء // بوابة المستوى
  if (item.subKey && mod && !canAccessSubPage(mod, item.subKey)) → إخفاء   // بوابة الميزة
  ثم فلترة الأبناء؛ إذا فرغ الأبناء → إخفاء العنصر الأب
ثم: قسم بلا عناصر بعد الفلترة → إخفاء القسم بالكامل (سطر 756)
```

**ملاحظات مُتحقَّقة:**
- الفلترة تطابق بوابات التوجيه الثلاث في `App.tsx` (اتساق جيد — مصدر واحد للدوال عبر `AppContext`).
- **حقل `perm` غير مُطبَّق** في `filterItems` (السطور 737-739 تفحص module/minRoleLevel/subKey فقط)؛ يوجد مساعد `passesPermCheck` (سطر 729-730) لكنه غير مستدعى في الفلترة → يؤكد **VIS-003**.
- الإجراءات السريعة (Quick Actions) مفلترة بـ `minRoleLevel` فقط (سطر 1220-1221).

## 5. قواعد حوكمة القائمة (تُلزم أي عمل قادم)

1. **قائمة واحدة:** كل عنصر تنقّل يُضاف إلى `allNavSections` فقط. ممنوع منطق قائمة موازٍ في صفحة.
2. **كل عنصر له بوابة:** يجب أن يحمل العنصر `module` (أو `minRoleLevel` للعرضي) — لا عنصر بلا بوابة. (يمنع تكرار VIS-001 على مستوى القائمة.)
3. **الخدمة المساندة داخل السياق:** المسار الخادم يظهر كعنصر فرعي ضمن قسم المسار القائد، لا كقسم مستقل. (يُفصَّل في `PATH_LEADER_SERVICE_MATRIX` و`SERVICE_MODE_VISIBILITY`.)
4. **لا عنصر لميزة غير مفعّلة:** بعد بناء حكم الاشتراك (MENU-003)، يُخفى العنصر إن لم تُفعَّل ميزته للمنشأة.
5. **العربية أولًا:** كل `label` بالعربية الواضحة لغير التقني؛ المصطلحات تتبع `ARABIC_BUSINESS_TERMS`.
6. **لا تكرار:** قبل إضافة عنصر، يُتحقَّق أن المسار غير مُمثَّل بعنصر آخر (مرجع: `registry.ts`).
