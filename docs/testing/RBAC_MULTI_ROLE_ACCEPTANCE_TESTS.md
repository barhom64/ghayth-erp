# اختبارات قبول الأدوار المتعددة — RBAC_MULTI_ROLE_ACCEPTANCE_TESTS

> المرحلة 7 — تنفيذ اختبارات **#1413 §16** · 2026-05-29 · فرع `claude/ghaith-foundation-audit-wdIUf`
> **المبدأ:** اختبار من الواجهة لنموذج المستخدم متعدد الأدوار. كل اختبار يتحقق من الصلاحية + النطاق + تسجيل الصفة.

---

### اختبار 1 — إنشاء سريع: موظف + حساب + دور واحد
- **إذن:** يُنشأ الثلاثة بمعاملة واحدة؛ الملخص العربي يعرض ما يستطيع.

### اختبار 2 — إنشاء سريع: موظف + حساب + 3 أدوار
- **إذن:** ثلاثة أدوار مسندة بنطاق لكل دور؛ الصلاحيات النهائية = اتحادها.

### اختبار 3 — موظف بثلاثة أدوار محدودة
أحمد: (إنشاء مالي فقط) + (عرض أسطول فقط) + (موظف ذاتي).
- **يستطيع:** إنشاء مصروف، عرض الأسطول، طلب إجازته.
- **لا يستطيع:** اعتماد المصروف، تعديل مركبة، رؤية رواتب الآخرين.
- **الأثر:** كل فعل يُسجَّل بالصفة المستخدمة.

### اختبار 4 — إنشاء + اعتماد في نفس المسار (SoD)
- **إذن:** إن منعت السياسة → منع باعتماد ما أنشأ برسالة عربية واضحة؛ إن سُمح → يُسجَّل في التدقيق.

### اختبار 5 — مدير فرع مكة
- **إذن:** يرى مكة فقط؛ لا يرى جدة/الرياض (نطاق branch).

### اختبار 6 — مدير مالي على مستوى الشركة
- **إذن:** يرى كل المالية؛ لا يستطيع تعديل سياسات HR.

### اختبار 7 — مسؤول عمرة
- **إذن:** ينشئ موسمًا/وكيلًا/معتمرًا؛ لا يرى إعدادات مالية متقدمة إلا بدور مالي.

### اختبار 8 — موظف منتهية خدمته
- **إذن:** يُعطَّل/يُقيَّد حسابه حسب السياسة دون حذف الموظف.

---

## التحقق المشترك في كل اختبار
1. الصلاحيات النهائية صحيحة (Effective Permissions Viewer).
2. "لماذا يستطيع/لا يستطيع" يُجيب بالعربية (Permission Explainer).
3. التعارضات الخطيرة تظهر (Role Conflict Analyzer).
4. **الدور المستخدم يظهر في سجل التدقيق** (RBAC-001 — معيار حاسم).
5. تبديل الصفة يغيّر الظهور والصلاحيات فورًا.
