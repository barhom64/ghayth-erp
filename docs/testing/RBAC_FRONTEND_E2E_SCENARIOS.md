## سيناريوهات الظهور

| # | السيناريو | المتوقّع |
|---|---|---|
| E1 | موظف عادي يفتح القائمة | يرى بوابة الموظف فقط؛ لا مالية/عمرة/نقل/إعدادات |
| E2 | محاسب يفتح المالية | يرى ما يملك صلاحيته؛ زر "اعتماد" مخفي |
| E3 | مدير فرع مكة يفتح الموظفين | بيانات مكة فقط؛ لا فروع أخرى |
| E4 | مسؤول عمرة يفتح القائمة | يرى العمرة كقسم قائد؛ لا أقسام خوادم مستقلة |
| E5 | **توجيه مباشر لمسار umrah بلا دور** | يُمنَع (يتحقق من إصلاح VIS-001) |
| E6 | فتح كيان: التعليقات/المرفقات | تظهر كتبويبات في السياق لا كأقسام (SERVICE_MODE_VISIBILITY) |
| E7 | زر "اعتماد" على فاتورة `draft` | مخفي (الحالة لا تسمح — ACTION_VISIBILITY_RULES) |
| E8 | زر "اعتماد" على فاتورة `submitted` لمن يملك approve | ظاهر |
| E9 | تبديل الصفة من "محاسب" إلى "موظف" | القائمة والصلاحيات تتغيّر فورًا |
| E10 | ميزة غير مفعّلة للمنشأة | لا تظهر إطلاقًا (VISIBILITY_ENGINE_SPEC) |
