# اختبارات تجربة المستخدم غير التقني — NON_TECHNICAL_USER_EXPERIENCE_TESTS

> المرحلة 7 — **Ghaith Operating Foundation** (#1418) · 2026-05-29 · فرع `claude/ghaith-foundation-audit-wdIUf`
> **المبدأ:** المستخدم غير التقني ينجز عمله دون شرح تقني. لو احتاج شرحًا تقنيًا فالتصميم فشل.

---

## منهجية
شخصيات غير تقنية (موظف، محاسب مبتدئ، مدير فرع، مسؤول عمرة) تنفّذ مهام واقعية بلا تدريب تقني، ونقيس: الإنجاز، الزمن، نقاط الحيرة.

## سيناريوهات

| # | الشخصية | المهمة | معيار النجاح |
|---|---|---|---|
| N1 | موظف | طلب إجازة | ينجزها من بوابته بلا مساعدة؛ يرى حالة الطلب |
| N2 | محاسب مبتدئ | تسجيل مصروف | يجد الزر بالعربية، يفهم رسالة "أُرسل للاعتماد" |
| N3 | محاسب | يحاول اعتماد مصروفه | يفهم رسالة المنع العربية (SoD) دون حيرة |
| N4 | مدير فرع | اعتماد إجازة موظفه | يجدها في مركز قراراته، يعتمد، يرى الأثر |
| N5 | مسؤول عمرة | إنشاء مجموعة | ينجزها دون رؤية مصطلحات مالية/تقنية |
| N6 | مدير النظام (غير مبرمج) | إنشاء موظف + 2 دور | يستخدم الإنشاء السريع + القوالب، يفهم الملخص العربي |
| N7 | مدير | معرفة "لماذا لا يستطيع X فعل Y" | المفسّر يجيب بالعربية |

## مقاييس
- **معدل الإنجاز** بلا مساعدة ≥ هدف.
- **نقاط الحيرة** (مصطلح غير مفهوم، زر مفقود، رسالة غامضة) = 0 في الواجهة الأساسية.
- **الأثر مفهوم:** المستخدم يعرف ماذا حدث بعد كل إجراء.

## القرار
- يربط `USER_JOURNEY_USABILITY_SCORECARD` (التقييم الكمّي) و`ARABIC_UX_LANGUAGE_ACCEPTANCE` (اللغة).
- أي حيرة تقنية = عيب تجربة يُسجَّل ويُعالَج.
