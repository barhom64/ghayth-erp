# FINANCE_OPERATION_CONTEXT_MATRIX.md

مصفوفة سياق العملية المالية الموحّد (FinanceOperationContext).

## التعريف

كل عملية مالية في غيث تُوصَف بسياق موحّد يحمل:

| الحقل | الوصف |
|---|---|
| `operationType` | نوع العملية (expense, receipt, payment, invoice, transfer, custody, …) |
| `party` | الطرف (عميل/مورد/موظف/جهة) |
| `moneySource` / `moneyDestination` | حساب مصدر/وجهة المال + usage |
| `paymentMethod` | طريقة الدفع/القبض (تُطابَق مع usage) |
| `allocationTarget` | هدف الربط (انظر FINANCE_ALLOCATION_TARGETS.md) |
| `operationalEffect` | الأثر التشغيلي (تذكرة صيانة، تحديث عهدة، …) |
| `journalEntry` | القيد المحاسبي الناتج |
| `affectedReports` | التقارير المتأثرة |
| `overrideReason` | سبب أي تجاوز يدوي |

## المصفوفة (عملية ← مصدر/وجهة ← قيد ← أثر ← تقرير)

| العملية | المصدر/الوجهة المسموح | القيد (مبسّط) | الأثر التشغيلي | التقرير المتأثر |
|---|---|---|---|---|
| مصروف نقدي | cash_box | مدين مصروف / دائن صندوق | حسب allocationTarget | المصروفات، CC P&L |
| مصروف بنكي | bank | مدين مصروف / دائن بنك | — | المصروفات، التدفق النقدي |
| مصروف من العهدة | custody | مدين مصروف / دائن عهدة | تخفيض رصيد العهدة | العهد، المصروفات |
| سند قبض | cash_box/bank | مدين نقد/بنك / دائن ذمم عميل | تخفيض ذمم العميل | المقبوضات، كشف العميل |
| سند صرف | cash_box/bank | مدين ذمم مورد / دائن نقد/بنك | تخفيض ذمم المورد + WHT | المدفوعات، كشف المورد |
| فاتورة مبيعات | receivable | مدين ذمم / دائن إيراد + ضريبة مخرجات + COGS | خصم مخزون | الفواتير، الإيرادات، VAT |
| فاتورة مورد | payable | مدين مصروف + ضريبة مدخلات / دائن ذمم مورد | totalSpend + budget | المشتريات، VAT |
| عهدة | custody | مدين عهدة / دائن نقد | فتح/زيادة عهدة | العهد |
| تحويل بين حسابات | حسب نوع الطرفين | مدين وجهة / دائن مصدر | — | الخزينة |
| قيد يدوي | أي (بصلاحية) | حسب البنود | — | دفتر الأستاذ |

## أين يُستخدم

- Backend: `financeOperationContext.ts` يبني السياق ويتحقق منه قبل الترحيل.
- Frontend: `FinanceOperationContextPanel` يعرض المصدر/الوجهة + هدف الربط
  + معاينة القيد، ويُستهلك في كل صفحات الإنشاء المالية.

## معايير القبول المرتبطة

- كل عملية مالية لها سياق واضح.
- كل ربط له أثر (قيد/تذكرة/سجل/تقرير).
- أي override يظهر في Audit وتقرير خاص.
