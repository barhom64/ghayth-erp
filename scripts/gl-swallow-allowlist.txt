# retired. The check (check-gl-swallow.mjs) now recognises that a GL-error catch
# which does NOT rethrow is still SAFE when the wrapped post is GUARANTEED to
# record the failure — i.e. it routes through createGuardedJournalEntry
# (businessHelpers.ts), which INSERTs into financial_posting_failures (drained
# by the postingFailureRetry cron) before rethrowing. Every previously-baselined
# file posts through a guardTable path, so the swallowed rethrow loses nothing.
#
# Live-proven by tests/integration/glGuardedFailureRecording.dynamic.test.ts: a
# forced guarded-post failure lands a financial_posting_failures row.
#
# Add a `routes/<file>.ts` / `lib/<...>.ts` line here ONLY if a genuinely
# UNGUARDED GL post (raw createJournalEntry, no recording) must be retained —
# the guard still fails on that. Real fixes (rethrow, or route through a
# guardTable post) are preferred over an allowlist entry.
