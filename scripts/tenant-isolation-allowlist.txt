# tenant-isolation-allowlist.txt — FND-013 triage baseline.
# Each line: routes/<file>.ts:<table> — a static read/write of a tenant table
# that lacks a literal "companyId" predicate in the SAME statement. The guard
# fails on any NEW offender not listed here.
#
# TRIAGE STATUS (2026-06-15, FND-013 review against the live head-of-main DB):
# every entry below was read end-to-end in its handler. NONE is an exploitable
# cross-tenant leak. Each is one of:
#   [GLOBAL]  — legitimately reads across companies (platform/system infra,
#               anonymous webhook with no session, public portal, or a
#               deliberately global uniqueness/template lookup). Adding a
#               companyId predicate would BREAK intended behaviour.
#   [TIGHTER] — already scoped tighter than companyId, by the caller's own
#               identity (userId/employeeId/activeAssignmentId) or by a parent
#               id that the SAME handler validated against scope.companyId
#               upstream. Cross-tenant leakage is not reachable. A cheap
#               defense-in-depth companyId predicate could be added per-track
#               but is not required for correctness.
# The one genuine HARDEN found (properties.ts maintenance_requests stats query)
# was fixed in-code (companyId predicate added) and its line removed below.
# Whole-file exception: routes/<file>.ts (no colon). Comments start with #.
