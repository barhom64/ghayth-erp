# لقطة حالة نظام غيث

> **تاريخ اللقطة: 18 يونيو 2026 — محدَّثة جزئيًا 21–22 يونيو 2026** (تحقّق فجوات السلامة المالية، اكتمال درج الإنشاء، توحيد الإدخال المالي، مسح المسارات 9–14، إغلاق بنود معمارية: RBAC حوكمة + توحيد مساري CRM + بُعدا fleetEngine وجرد دورة العد؛ FX revaluation الخيار أ مُنفَّذ #2873 + إصلاح خلل idempotency #2897 بتحقّق تشغيلي؛ لوحة الصدق مُكمَّلة: قياس القيود اليتيمة #2874 + بوابة إقفال حاجبة B3 #2902). هذه نقطة زمنية ثابتة من ذاكرة إدارة المشروع، **لا تتحدث تلقائيًا.** عند أي تعارض، الكود الفعلي وحالة الـ PRs الحية هي الحقيقة. حدّث هذا الملف يدويًا بعد كل تغيير جوهري.

## البنية التقنية الجارية (تطوّرت عن المواصفات الأصلية)

pnpm monorepo: `api-server` (Node.js/Express + PostgreSQL + Drizzle ORM) — `ghayth-erp` (React 19 + TypeScript + Tailwind) — `ghayth-erp-deck`. الهجرات (migrations) ملفات مرقّمة في `src/migrations`. النطاق: 3500+ commit، 3500+ ملف، 14 نطاقًا وظيفيًا، 300+ هجرة، 600+ صفحة، 100+ ملف مسار backend، 1200+ نقطة API.

## حالة المسارات

- **المالية (S7):** PR #1997 معتمد (معاينة فاتورة تعرض حسابات AR/إيراد/VAT — للقراءة فقط، لا تغيير ترحيل). محرك إطفاء التأمين معتمد للبناء: Engine A ينشئ جداول الاعتراف عند الرسملة؛ إطفاء شهري idempotent عبر `postJournalEntry`. تأمين العقار والتأمين الطبي بلا محركات ترحيل GL بعد.
- **الموارد البشرية:** Wave 0 مكتملة (كل النقاط مبوّبة بـ `authorize()`، اختبارات RBAC ratchet، ترتيب H0→H8). Wave 1B بدأت. 4 قوالب أدوار مكسورة: `hr_admin` بلا تعريف في الهجرات؛ `attendance_officer` و`discipline_officer` و`performance_reviewer` بلا `rbac_role_grants`.
- **النقل:** الفرع النظيف معتمد — commit السقالة أُسقط، الهجرة 282 مؤكدة (لا تمس القيود).
- **الأملاك:** PR #1999 معتمد (contractType موحّد، هجرة idempotent 286). الـ PRs الماسة للقيود 4/5/6 معلّمة لتحقق خاص.
- **المستودعات:** قلق ProductSelect العابر للنطاقات حُلّ (وكيل المستودعات عدّل `invoices-create.tsx` المملوك للمالية)؛ الإصلاح مؤكد (`sellPrice` هو الحقل الصحيح).
- **المشاريع:** فشلان صامتان أُصلحا — أخطاء 500 في إكمال المرحلة (كانت تعطّل فوترة المعالم)، وترحيل WIP ميت بسبب مفاتيح GL غير مزروعة.
- **العمرة:** PRs #1920 و#1931 و#1995 جاهزة للدمج. PRs #1996 و#1998 فيها bug واجهة حقيقي (`formatDateAr` بدل `formatUmrahDate` — يخالف تفضيل التقويم الهجري) — مأذون بإصلاحه، لا بتجاوز الحارس.

## فجوات السلامة المالية

> **تحقّق 2026-06-21 على الكود الفعلي:** ثلاث من الفجوات الأربع أُغلقت، والرابعة كانت لقطة خاطئة. لا عمل دفتر آمن متبقٍّ.

- ✅ **مُغلقة** — إطفاء أقساط التأمين: `insuranceEngine.ts` + `prepaidAmortizationEngine.ts` يوفّران محرك اعتراف + إطفاء شهري idempotent (DR مصروف / CR 1172) لنوعَي التأمين العقاري والطبي عبر `prepaid_amortization_schedules` + `prepaid_amortization_postings`.
- ✅ **لا فجوة (تصحيح لقطة)** — `propertiesEngine.ts` يستخدم حساب **2150** الصحيح (مستحقات صيانة، `property_maintenance_payable`)، لا 2160. حساب 2160 محفوظ للإيراد المؤجّل للعملاء.
- ✅ **مُغلقة** — القيد اليدوي: `finance-hardening.ts` يفرض سير `draft→review→approve→post` + maker-checker (المنشئ لا يراجع/يعتمد قيده) + `requireMinLevel(70)` + فحص توازن وفترة مفتوحة.
- 🟢 **مُغلق تشغيليًا** — تعبئة الأبعاد: `journalLineDimensionalEnricher.ts` + المحركات (umrah/fleet/hr/properties/finance) تمرّر أبعادها على كل السطور. أُغلقت فجوتا فقد بُعد صامتتان مع assertion tests: `fleetEngine.postViolationPaymentGL` (vehicleId على سطر النقد، #2863) و`post-cycle-count-journal` (warehouseId عبر dimensionJson، #2866). **FX revaluation (الخيار أ مُنفَّذ، #2873):** القيد المجمّع القديم كان **مرفوضًا فعلًا** عند الترحيل (إنفاذ الأبعاد على 1131/2111)، فصار مفصّلًا لكل كيان (سطر AR لكل عميل بـclientId، AP لكل مورد بـvendorId، + سطرَي مكسب/خسارة إجماليين) مع assertion tests. **خلل سابق كشفه التحقّق التشغيلي (#2897):** فحص idempotency كان يستعلم عمود `revaluationDate` الوهمي (الجدول يستخدم `period`) → الـendpoint كان يرمي 500 لأي طلب؛ أُصلح وتُحقّق E2E (قيد فعلي: AR 250 بـclientId، متوازن، اجتاز الإنفاذ). **متبقٍّ (تنظيف):** `ensureFxTables` في finance-algorithms.ts يُنشئ `fx_revaluations` بمخطط قديم مضلِّل (CREATE IF NOT EXISTS مظلَّل، لا يضرّ)؛ والمسار المكتبي `lib/fx/post-revaluation-journal.ts` غير موصول — توحيدهما بند مستقل. المصادر بلا بُعد طبيعي (قيد يدوي/تسوية بنكية/إقفال) = null مقبول بحكم التصميم.

## مبادئ متراكمة (إضافة للدستور)

- **فهرسة الحقول المشفّرة:** لا يمكن فهرسة الحقول المشفّرة مباشرة (AES-GCM IVs عشوائية). استخدم نمط Encrypted+Hash: AES-256-GCM للتخزين، HMAC-SHA256 للبحث.
- **Mock-First للتكاملات الخارجية:** 3 شروط مسبقة قبل بناء المحوّلات الحقيقية.
- **ProductSelect مشترك واحد** يُبنى مرة (المستودعات) ويُعاد استخدامه — لا تكرار.
