# تسليم تطبيق الجوال — غيث ERP

يوضّح هذا المستند كيفية تشغيل تطبيق الجوال وتسليمه للمستخدمين عبر مسارين:

1. **رابط ويب / PWA (متاح الآن)** — يعمل فوراً، متصل بالخادم.
2. **ملف APK لأندرويد (لاحقاً)** — يتطلب حساب Expo/EAS خاص بكم.

---

## 1) رابط الويب / PWA (جاهز الآن)

بعد دمج هذا التغيير ونشره على الخادم، يصبح التطبيق متاحاً على:

```
https://erp.door.sa/mobile/
```

- نفس النطاق الذي يخدم الـ API (`https://erp.door.sa/api`) → الاتصال بالخادم يعمل تلقائياً (نفس الأصل، بدون مشاكل CORS).
- يفتح في أي متصفح على الجوال (Android / iOS) أو على سطح المكتب.
- قابل للتثبيت كتطبيق (PWA): على أندرويد عبر Chrome → قائمة (⋮) → **«إضافة إلى الشاشة الرئيسية / تثبيت التطبيق»**، وعلى iOS عبر Safari → زر المشاركة → **«إضافة إلى الشاشة الرئيسية»**. يظهر بعدها كأيقونة تطبيق مستقلة بملء الشاشة (standalone) بواجهة عربية RTL.

### كيف يُبنى ويُخدم
- البناء: `pnpm --filter @workspace/ghayth-mobile run build`
  - يُصدِّر تطبيق ويب أحادي الصفحة (SPA) إلى `static-build/` عبر `expo export --platform web`.
  - يحقن مسار القاعدة `/mobile/` ونطاق الـ API (`EXPO_PUBLIC_DOMAIN`، الافتراضي `erp.door.sa`).
  - يضيف طبقة PWA: `manifest.webmanifest` (عربي/RTL، standalone، أيقونات 192/512/maskable) + service worker بسيط + معالجة `index.html`.
- الخدمة: `pnpm --filter @workspace/ghayth-mobile run serve`
  - يخدم `static-build/` مع SPA fallback (أي مسار غير ملف → `index.html`)، و`/status` للفحص الصحي، ورؤوس التخزين المؤقت الصحيحة.

> ملاحظة: معاينة التطوير (`expo start`) لم تتغيّر — مسار القاعدة `/mobile/` يُحقن وقت بناء الإنتاج فقط ثم يُستعاد `app.json`.

---

## 2) ملف APK لأندرويد (الخطوة اللاحقة)

التطبيق مبني على Expo، وبناء APK أصلي يتم عبر **EAS Build** من Expo. هذا يتطلب موارد خاصة بكم ولا يمكن تنفيذه نيابةً عنكم من هنا.

### ما المطلوب منكم لإنتاج APK
1. **حساب Expo** (مجاني للبدء): أنشئوه على https://expo.dev.
2. **رمز وصول Expo (Access Token)**: من لوحة تحكم Expo → Account → **Access Tokens**. سيُحفظ كسر (secret) باسم مثل `EXPO_TOKEN`.
3. (اختياري لاحقاً للنشر على Google Play) حساب **Google Play Console** (رسوم تسجيل لمرة واحدة) ومفتاح توقيع التطبيق.

### الخطوات (تُنفَّذ بعد توفير ما سبق)
1. إضافة `eas.json` بملف تعريف بناء `apk` (نوع build: `apk` بدل `aab` لتثبيت مباشر خارج المتجر).
2. ضبط `android.package` في `app.json` (مثل `sa.door.ghayth`).
3. تثبيت/تشغيل: `eas build -p android --profile preview` (ينفّذ على خوادم Expo، لا يحتاج Android Studio محلياً).
4. عند الانتهاء يوفّر Expo رابط تنزيل مباشر لملف `.apk` يمكن مشاركته للتثبيت اليدوي على أجهزة أندرويد (مع تفعيل «التثبيت من مصادر غير معروفة»).

### ملاحظات مهمة
- التطبيق الأصلي (APK) سيستهلك نفس واجهات الـ API على `erp.door.sa` — لا حاجة لتغيير الخادم.
- تدفّق النشر المُدار من Replit لتطبيقات Expo موجّه لنظام iOS فقط؛ لذا أندرويد يمرّ عبر EAS بحسابكم.
- لا يمكن تخزين رمز Expo أو تشغيل EAS تلقائياً دون موافقتكم وتزويدنا بالرمز.

**باختصار:** زوّدونا بحساب Expo + `EXPO_TOKEN`، وعندها نضيف `eas.json` ونوثّق أمر البناء لإصدار APK قابل للتنزيل.
